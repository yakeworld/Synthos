# Synthos Project Quality Thresholds

> 闭环质量标准：所有维度达标 → 项目闭环完成 ✅
> 最后更新: 2026-05-23

## 六维质量标准

| 维度 | 权重 | 达标阈值 | 当前分 | 状态 |
|:-----|:-----|:---------|:------|:----:|
| D1 基础设施完整性 | 15% | ≥0.95 | 0.92 | 🟡 |
| D2 认知原子健康 | 25% | ≥1.0 | 0.90 | 🟡 |
| D3 Evolution引擎就绪 | 20% | ≥0.95 | 0.90 | 🟡 |
| D4 项目文档一致性 | 15% | ≥0.95 | 0.88 | 🟡 |
| D5 代码/文件整洁 | 10% | ≥0.90 | 0.85 | 🟡 |
| D6 外部可接入性 | 15% | ≥0.90 | 0.88 | 🟡 |

**综合加权分**: 0.89（目标 ≥0.95）

## 评分标准

### D1 基础设施完整性
- git 管理 ✅
- CI 管线 ✅
- LICENSE ✅
- README 中英分离 ✅

### D2 认知原子健康
- 所有原子技能文件完整
- Golden 测试案例存在
- 原子信任度 ≥ 0.95

### D3 Evolution引擎就绪
- 引擎文件在项目内可访问
- evolution-state.json 与引擎版本一致
- 所有引用的版本号同步

### D4 项目文档一致性
- README.md 数据与实际一致
- evolution-state.json 版本号与实际一致
- SKILL.md 版本号与实际一致
- 三处不矛盾

### D5 代码/文件整洁
- 无大量冗余中间文件
- archive/ 目录干净
- 无 .bak 残留文件

### D6 外部可接入性
- setup 指引存在
- 新用户 onboarding 路径清晰
- agent 可独立使用

## 闭环退出条件

> **全部 6 维 ≥ 达标阈值** → 项目闭环完成 ✅

首次达标后，切换到下一个项目。
