# Synthos 原则体系 v1.0-draft-r2

> 地位：**宪法层**。约束所有层的所有组件。原则（P0–P3）一旦定版，不再修改；实施细则可通过受控变更流程演化。

---

## 第一部分 · 四条原则（规范性正文）

### P0 · 证据可溯（Evidence Traceability）

**陈述**：每个产出是证据链，不是结论。所有产出必须可追溯、可共享、有生命周期。

**约束对象**：所有层。

**具体要求**：
1. 每个原子的输出必须携带 `evidence_chain` 字段，引用其输入来源。
2. 路径决策（路由器的选择、pipeline 的跳转）本身是证据，须记录到 CallGraph。
3. **空结果是结构化证据，不是无数据**。检索返回 0 条时，EvidenceChain 必须记录：数据库清单、查询字符串、同义词展开方案、时间戳、排除规则。"没找到"≠"不存在"，两者的区别只能通过证据本身确立。
4. 证据链可追溯至外部标识符（DOI、URL+fetch_time、数据集版本号等）。

**显式不保证**：外部源的持久可达性。P0 保证"我们记录了当时拿到什么"，不保证"未来仍能从该源取回相同内容"。

---

### P1 · 原子可复现（Atomic Reproducibility）

**陈述**：原子层是稳定面。稳定的定义依原子类型而异。旧版本原子保留可调用——复现过去的结果不需要"升级"。

**约束对象**：原子层。

**原子分为两类**（见第二部分），两类的"可复现"含义不同：

| 原子类型 | 可复现的含义 | 保证机制 |
|---------|------------|---------|
| 机械原子 | **查询可复现**：同一 query + 同一参数 → 同一请求 | 代码哈希 + 参数锁定 + 外部依赖显式声明 |
| 认知原子 | **语义可复现**：同一输入 + 同一模型版本 → 等价结论（通过金标准测试集） | SKILL.md 哈希 + 模型ID+版本 + 金标准引用 + 通过阈值 |

**金标准的冷启动与演化规则**：
1. **优先检索**已有的外部金标准（领域公认测试集、benchmark、公开数据集）。
2. **无外部可借鉴**时，由原子作者（AI）提出自设金标准草案，经人类审批后冻结为 v0.1。自设金标准必须显式标注 `golden_set_origin: self_defined`。
3. 后续发现外部金标准时，触发重评估，并按受控变更流程决定是否替换。
4. **金标准与原子版本强绑定**：修改金标准 = 升原子版本。这是 P1 的硬约束，不可绕过。

**显式不保证**：
- 机械原子**不保证**外部源返回内容的稳定性（API 漂移、数据库更新在保证之外）。
- 认知原子**不保证**比特级输出一致（LLM 非确定性），只保证语义等价通过金标准。
- 两者都**不保证**跨版本可复现：要复现旧结果，必须调用旧版本原子。

---

### P2 · 稳定下沉 / 演化上浮（Stability Descends, Evolution Ascends）

**陈述**：稳定的能力下沉到原子层，演化的能力上浮到组合层。两层通过 I/O 契约解耦。**原子集合的演化是受控的，不是自由的**。

**约束对象**：架构。

**具体要求**：
1. 原子层暴露 `(atom_type, input_schema, output_schema, version, reproducibility_contract)`，组合层不关心原子内部实现。
2. 编排逻辑（pipeline 路径、重跑决策、条件分支）不得下沉到原子内部。典型反例：如果原子6的验证结果触发回到原子3重跑，这个回跳逻辑必须在组合层，不在原子6。
3. **机械原子**：可插拔工具箱，增删不触发宪法修订，由组合层按需调用。
4. **认知原子**：集合可演化，但每次增减须走**受控变更流程**：
   - AI 提出增减提案，必须包含：新原子的职责边界、与现有原子的非重叠性证明、input/output schema 草案、金标准方案（外部引用或自设草案）。
   - 人类审批。审批通过后记入修订历史，并更新 atom base 规范。
   - **一次任务中不得新增认知原子**。新增必须是跨任务的、显式的、留痕的——即"这个任务缺能力"不是增原子的理由，"多个任务都反复缺同一能力"才是。
5. 组合层的 pipeline 本就应该按课题演化，不受 P2 的稳定性约束。

**显式不保证**：组合层的稳定性。

---

### P3 · 人机分层（Human-Machine Layering）

**陈述**：默认审计（记录不等待），例外审批（不可逆动作必须人类显式确认）。

**约束对象**：交互层 / 组合层的产出交付边界。

**具体要求**：
1. 原子内部不承载 P3。检索、提取、关联、假设、论证、验证——全部自动完成并留痕。
2. P3 落在**组合层的对外交付点**：提交 NSFC、发布报告、写入医疗/临床系统、触发对外通信、写入不可回滚的数据库。
3. "不可逆动作"的判定规则必须显式声明，不得隐式。组合层 pipeline 的每一步须标注 `requires_human_approval: bool`。
4. **受控变更流程**（见 P2 第4条、P1 金标准规则）中的"人类审批"也属于 P3 的作用范围：认知原子增减、金标准冻结、原子版本升级——均为不可回滚操作，须人类显式确认。

**显式不保证**：对"可逆动作"的审批。频繁弹窗会让审批疲劳并失效。

## 第二部分 · 原子分类（P1 / P2 的结构化基础）

### 机械原子（Mechanical Atoms）
- **特征**：确定性代码执行，无推理成分。
- **稳定性**：代码哈希 + 参数锁定。
- **示例**：知识获取（API 调用）、格式转换、数据下载、文件读写。
- **集合性质**：可插拔工具箱，按需增删，不触发宪法修订。

### 认知原子（Cognitive Atoms）
- **特征**：Agent + SKILL.md 驱动的推理过程。
- **稳定性**：SKILL.md 冻结 + 模型版本冻结 + 金标准测试通过。
- **当前集合**（v1.0 起点，覆盖研究认知闭环）：
  - 路由器（最短链编排）
  - 知识提取
  - 关联发现
  - 观点生成
  - 论证表达
  - 观点验证
- **集合性质**：可演化，但演化受 P2 第4条的受控变更流程约束。

> 注：路由器归类为**认知原子**。其 `output_schema` 必须包含路径决策的 rationale（走了哪条链、跳过了哪些原子、为何跳过）——这是 P0 对路由的具体要求。

## 第三部分 · 契约形状（P1 / P2 的传导结果）

AtomicSkill 契约的最小形状：

```
AtomicSkill = (
    atom_type           : {MECHANICAL | COGNITIVE}
    input_schema        : JSONSchema
    output_schema       : JSONSchema   # 必须包含 evidence_chain 字段
    version             : SemVer
    reproducibility_contract : ReproContract
)
```

`ReproContract` 依 `atom_type` 分为两个变体：

```
ReproContract.MECHANICAL = {
    code_hash          : str
    param_lock         : dict
    external_deps      : list[ExternalDep]
    guarantees         : "query_reproducibility"
    non_guarantees     : "external_content_stability"
}

ReproContract.COGNITIVE = {
    skill_md_hash      : str
    model_id           : str
    model_version      : str
    golden_set_ref     : URI
    golden_set_origin  : {EXTERNAL | SELF_DEFINED}    # 金标准来源
    pass_threshold     : float
    guarantees         : "semantic_reproducibility"
    non_guarantees     : "bitwise_output_equality"
}
```

**关键设计**：`non_guarantees` 是契约的一等公民。契约的边界本身是证据（P0 的自洽递归）。

## 第五部分 · 宪法稳定性条款

1. **原则冻结**：P0–P3 一旦定版，不再修改。这是 Synthos 的根本承诺。
2. **细则可演化**：原则的实施细则（认知原子集合、金标准流程、契约 schema 的具体字段）可通过受控变更流程演化。
3. **判定依据**：变更是否改变"原则 × 组件 约束矩阵"的单元格含义——
   - 若**未改变**，则是细则演化，走受控变更流程（AI 提案 + 人类审批）。
   - 若**改变了**，则是原则修订——**禁止**。
4. **受控变更流程的产物**：每次变更须产出变更记录，包含：提案内容、非重叠性/必要性论证、影响的组件清单、人类审批人、审批时间戳。变更记录本身是 P0 证据。
