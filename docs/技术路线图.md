## 二、输入输出数据流

### 科研主流程

```
用户 Query
  │
  ▼
[路由器] 判断任务复杂度
  │
  ├─→ [原子1: 知识获取]
  │    输入: 研究主题/关键词/研究问题
  │    输出: raw_papers[], pdf_paths[], citations[], teaching_sources[]
  │    数据源: PubMed, Semantic Scholar, Crossref, OpenAlex
  │           + 73个教学来源 (PDF, Google Docs, Markdown)
  │
  ▼
[原子2: 知识提取]
  │    输入: pdf_paths[], raw_papers[], teaching_sources[]
  │    输出: extracted_knowledge[], knowledge_graph, methods[], datasets[]
  │
  ▼
[原子3: 关联发现]
  │    输入: extracted_knowledge[], knowledge_graph
  │    输出: associations[], research_gaps[], contradiction_map[]
  │
  ▼
[原子4: 观点生成]
  │    输入: associations[], research_gaps[]
  │    输出: hypotheses[], rationale[], novelty_score[], crsp_dm_plan[]
  │    (结合 CRISP-DM 方法论生成模型构建方案)
  │
  ▼
┌──────────────────────────────────────┐
│              [原子5+6]                 │
│  分支A: 论证表达 ──→ [原子5]          │
│         输入: hypotheses[], rationale[] │
│         输出: sections[], arguments[], references[], tripod_report[]
│         (结合 TRIPOD+AI 标准生成报告) │
│                                          │
│  分支B: 观点验证 ──→ [原子6] ←─ 双向迭代 │
│         输入: hypotheses[], rationale[] │
│         输出: verification_results[], weaknesses[], confidence[]
│         (结合 CRISP-DM 评估 + 概率校准) │
└──────────────────────────────────────┘
  │
  ▼
最终输出: 综述论文 / 基金申请书 / 研究报告 / 临床预测模型报告
```

### 教学主流程

```
学生/教师 Query
  │
  ▼
[路由器] → 判断教学场景 → 匹配教学来源
  │
  ├─→ [原子1: 知识获取]
  │    输入: 学生问题/学习需求
  │    输出: 相关教案, 数据集, 方法论文档, 学术论文
  │    (从73个教学来源中精准匹配)
  │
  ▼
[原子2: 知识提取]
  │    输入: 教学来源
  │    输出: 结构化教学知识, 关键概念, 实践步骤
  │
  ▼
[原子3: 关联发现]
  │    输入: 结构化教学知识
  │    输出: 知识点关联, 学习路径建议, 实践项目推荐
  │
  ▼
[原子4: 观点生成]
  │    输入: 关联知识
  │    输出: 学习方案, 实践建议, 科研方向建议
  │
  ▼
[原子5: 论证表达]
  │    输入: 学习方案
  │    输出: 个性化学习报告, 学习进度追踪
  │
  ▼
[原子6: 观点验证]
  │    输入: 学习方案
  │    输出: 学习效果评估, 改进建议
  │
  ▼
最终输出: 个性化学习方案 + 学习效果评估
```

### 进化触发条件

1. **定时触发**: 每 1000 秒执行一次完整评估
2. **异常触发**: 某原子质量 < 阈值时立即触发
3. **任务触发**: 每次任务执行后记录性能数据
4. **外部触发**: 检测到新技能/新教学来源时触发吸收流程

## 四、技术栈

## 五、CRISP-DM 映射

| CRISP-DM 阶段 | 对应 Synthos 原子 | 功能 |
|--------------|------------------|------|
| 1. 业务理解 | 原子1 (知识获取) | 理解医学问题背景 |
| 2. 数据理解 | 原子2 (知识提取) | 提取数据特征和分布 |
| 3. 数据准备 | 原子2 (知识提取) | 特征选择、数据清洗 |
| 4. 建模 | 原子4 (观点生成) | 生成假设和模型方案 |
| 5. 评估 | 原子6 (观点验证) | 证伪检验、概率校准 |
| 6. 部署 | 原子5 (论证表达) | 生成报告、TRIPOD输出 |

## 六、Agent Skills 规范合规

所有 7 个技能（6 原子 + 1 路由器）完全符合 Agent Skills 规范：

- ✅ **目录命名**: 英文 snake_case（如 knowledge-acquisition）
- ✅ **frontmatter**: name 字段匹配目录名
- ✅ **description**: 325-464 字符，含具体关键词
- ✅ **license**: MIT
- ✅ **metadata**: author, version, created
- ✅ **allowed-tools**: delegate_task, Read, Write, Execute
