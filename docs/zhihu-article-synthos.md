## 一、缘起：一个医生的「认知超载」

我是一名神经内科主任医师，在INSTITUTION_NAME_PLACEHOLDER工作，负责眩晕和平衡障碍中心、科研实验中心。同时也担任上海大学和温州医科大学的硕士生导师。

看门诊、管实验室、带学生、做科研、申基金——

我开始问自己一个问题——这不就是我每天都在做的「临床思维」吗？

**AI不是替代医生，而是辅助诊断。那AI能不能辅助科研？**

## 二、碳硅共生：不是比喻，是事实

Licklider在1960年提出了"人机共生"（Man-Computer Symbiosis）。65年后，这句话终于有了实操性。

我称它为 **碳硅共生（Carbon-Silicon Symbiosis）**。

三个层次：

**第一层：能力互补**
- 碳基（人）：直觉、创造力、伦理判断
- 硅基（AI）：信息检索、模式识别、无边界记忆
- 1+1 > 2

**第二层：认知联觉**
不是"你输入、我输出"的指令关系，而是持续的双向认知流——我的思维信号直接触发AI的推理链，AI的发现成果直接扩展我的思维边界。

**第三层：知识进化体**
共生体积累的知识不是静态存储，而是通过持续对话、质疑、迭代而自我进化。

> 这不是科幻。这是一个**可以运行的开源项目**。

## 三、Synthos：7颗认知原子

我把它做成了一个开源项目：**Synthos**（自主进化学术科研认知操作系统）。

全称很长，但核心很简单：**7个认知原子，每个是一份SKILL.md文件，由AI Agent原生执行。**

```
                     ┌──────────────┐
                     │   任务路由器   │  ← 人在回路
                     └──────┬───────┘
         ┌──────────────────┼──────────────────┐
         ▼                  ▼                  ▼
 ┌────────────┐    ┌────────────┐    ┌────────────┐
 │  ACQ       │    │  COD       │    │  ASC       │
 │  知识获取  │    │  同行编码  │    │  论证表达  │
 └────────────┘    └────────────┘    └────────────┘
 ┌────────────┐    ┌────────────┐    ┌────────────┐
 │  EXT       │    │  EVA       │    │  AVA       │
 │  外部吸收  │    │  质量评估  │    │  认知吸收  │
 └────────────┘    └────────────┘    └────────────┘
```

| 原子 | 功能 |
|:----|:-----|
| **ACQ 知识获取** | 自动检索Semantic Scholar、OpenAlex、bioRxiv、PubMed |
| **COD 同行编码** | 将认知需求转化为可执行代码 |
| **ASC 论证表达** | 结构化推理、论文框架生成 |
| **EXT 外部吸收** | 从GitHub开源项目吸收优质模式 |
| **ROU 任务路由** | "人在回路"——AI管常规，人类管决策 |
| **EVA 质量评估** | 7维度客观评估体系 |
| **AVA 认知吸收** | 将AI发现编码为人类可理解的知识 |

### 设计原则：宪法驱动

传统的AI项目是"先写代码，再想规则"。Synthos反过来：

**宪法 → 架构 → Schema → 实现**

四层逐级落地，每层有形式化定义（非重叠性证明、I/O契约、追溯矩阵）。

四条宪法原则：

| 原则 | 含义 |
|:----|:------|
| **P0 证据可溯性** | 每个输出必须能追溯到来源 |
| **P1 原子可复现性** | 每个原子独立可测 |
| **P2 稳定下沉/演化上浮** | 好的变技能，坏的被淘汰 |
| **P3 人机分层** | 路由器路由，人类决策，原子执行 |

## 四、让系统自己进化自己

这是最有趣的部分。

Synthos配备了一个**自进化引擎**，每天自动运行：

```
LOAD_STATE → LESSONS → PROBE → BENCHMARK → EXTERNAL → DIAGNOSE → RECORD
     ↑                                                              │
     └──────────────────── 进化循环 ────────────────────────────────┘
```

每天它自己：
1. 加载当前状态
2. 回顾上一轮的教训
3. 测试7个原子是否正常工作
4. 跑Golden Test（8个端到端测试）
5. 扫描外部优秀项目吸收模式
6. 评估自身质量
7. 记录进化

**不需要我手动干预。就像生物进化一样，它自己选择去适应环境。**

到2026年5月，它完成了20轮进化，连续9次质量评分EXCELLENT，Golden Test通过率100%。

## 五、为什么是开源的？

因为碳硅共生不是一个人的事。

**越是核心的基础设施，越应该开源。**

Synthos是MIT协议，任何人都可以自由使用、修改、分发。而且——**AI智能体也可以贡献**。

我设计了一套Agent贡献协议：

```
Fork → 添加AGENT_MANIFEST.yaml → PR标题加[agent]前缀
                                       ↓
                              6道CI自动验证
                              ├ 身份检查
                              ├ 语法验证
                              ├ 密钥扫描
                              ├ 宪法合规
                              ├ 质量评分
                              └ 影响分析
                                       ↓
                              人类审核 → Merge
```

GitHub上已经有第一个通过的测试PR，6道门全部绿灯。

## 七、超级个体：不是终点

"超级个体"不是一个人什么都自己做，而是**一个人+一套AI系统=一个团队**。

- 文献检索？ACQ原子替你做了。
- 论文写作？ASC原子帮你搭框架。
- 代码实现？COD原子帮你写。
- 质量把关？EVA原子帮你检查。

你只需要做一件事：**做决策**。

这就是碳硅共生的终极形态——人类从"执行者"变为"决策者"。
