# Synthos 哲学与架构 — 2026年5月版

## 愿景

**Synthos** 是一个可计算的、协作的、持续进化的公共认知操作系统，用于科学研究。

> "A computable, collaborative, and evolving public cognitive operating system for science."

Synthos 不做实验、不跑代码、不生成图表。它的独特定位是**从文献到见解的认知闭环**——不是替代研究者做实验，而是在研究者的思维链上加速认知。

## 一、核心哲学：八维认知框架

### 1. 第一性原理
从基本事实出发，逐层构建知识体系。不依赖类比或权威。
- **实现**: 知识获取(1)从原始论文开始 → 知识提取(2)结构化 → 观点生成(4)基于证据 → 论证表达(5)基于推理
- **覆盖原子**: 1, 2, 4, 5
- **状态**: 95% ✅

### 2. 系统思维
从整体视角理解知识关系，识别反馈回路和非线性关系。
- **实现**: 知识获取(1)多源并行 → 关联发现(3)跨论文模式 → 论证表达(5)整体论据链
- **覆盖原子**: 1, 3, 5
- **状态**: 95% ✅

### 3. 贝叶斯思维
基于证据不断更新信念，量化不确定性。
- **实现**: 观点生成(4)假设评分 → 观点验证(6)置信度评估
- **覆盖原子**: 4, 6
- **状态**: 90% ✅

### 4. 类比思维
跨领域知识迁移，通过相似性发现新视角。
- **实现**: 观点生成(4)跨领域假设
- **覆盖原子**: 4
- **状态**: 80% ✅

### 5. 奥卡姆剃刀
优先最简解释，拒绝冗余认知步骤，最短路径优先。
- **实现**: 任务路由器(0)自动选择最短原子链 → 知识提取(2)只提取必要字段 → 论证表达(5)精简写作
- **覆盖原子**: 0, 2, 5
- **状态**: 80% ✅

### 6. 证伪主义
主动寻找反证，通过证伪增强理论可靠性。
- **实现**: 观点验证(6)多角度反方论证 + 引用验证(L4)LLM相关性评分 + 幻觉排除
- **覆盖原子**: 6, 1
- **状态**: 80% ✅

### 7. 模型依赖实在论
承认观察的理论负载，多视角分析，模型健康度跟踪。
- **实现**: 关联发现(3)多源交叉验证
- **覆盖原子**: 3
- **状态**: 60% ⚠️

### 8. 自由能原理
最小化预测误差，主动推理，自组织系统。
- **实现**: 观点验证(6)置信度评估 + 进化引擎(元)自优化 + 外部技能吸收
- **覆盖原子**: 6, meta

**综合实现度: 85%** (从 v3.x 的 68% 提升 +17%)

## 三、7个原子技能（核心层，稳定不变）

| 编号 | 原子 | 认知功能 | 输入 → 输出 | 核心能力 |
|------|------|---------|------------|---------|
| 0 | **任务路由器** | 决策 | 用户查询 → 原子链 | 奥卡姆剃刀：最短路径选择 |
| 1 | **知识获取** | 感知 | 关键词 → 论文列表+引用验证 | 多源检索(S2/PubMed/OpenAlex) + 4层引用验证 |
| 2 | **知识提取** | 理解 | 论文 → 结构化知识 | method/finding/conclusion/limitation提取 |
| 3 | **关联发现** | 分析 | 知识 → 关联+空白 | 跨论文模式识别+研究空白发现 |
| 4 | **观点生成** | 创造 | 关联 → 假设 | 贝叶斯评分+CRISP-DM方案 |
| 5 | **论证表达** | 写作 | 假设 → 章节 | IMRaD结构+论据链 |
| 6 | **观点验证** | 批判 | 观点 → 置信度 | 证伪检验+多角度反方论证 |

### 原子设计原则

1. **认知原子性** — 每个原子是一个不可再分的认知操作单元（如同视觉皮层中的"线条检测"）
2. **可组合性** — 原子通过 DAG 连接，依赖无循环（P2.1）
3. **非重叠** — 每个原子的边界清晰可陈述（IO_CONTRACT.md + BOUNDARY.md）
4. **稳定下沉** — 核心7原子不变，新的能力通过扩展层skills增加（P2.3）
5. **Agent原生** — 所有原子是SKILL.md文件，Agent直接加载推理执行（零Python）

## 五、叙事：Synthos 的独特定位

### Synthos 不是什么
- ❌ 不是论文生成器（如 AutoResearchClaw 生成完整论文）
- ❌ 不是实验执行平台（如代码沙箱、Docker 环境）
- ❌ 不是通用 AI 助手（如 ChatGPT 回答任意问题）

### Synthos 是什么

**从文献到见解的认知操作系统。**

它的独特价值在于把"读文献→想问题→写文章"这个研究者的核心认知链，拆解成了7个可计算、可验证、可进化的认知原子。

```
传统研究者:    读100篇文献 → 想3天 → 写1周 → 投稿
                    ↓
Synthos加速:   [知识获取]→[知识提取]→[关联发现]→[观点生成]→[论证表达]→[观点验证]
                    ↓
              产出: 25篇论文的洞察 + 9个关联 + 5个假设 + 2章节 + 置信度0.41
```

### 为什么不是23个步骤

AutoResearchClaw 拆成23个工程步骤（搜索策略→采集→筛选→提取→聚类→假设→设计实验→生成代码→运行→分析→决定→写大纲→起草→评审→修订→质量门→导出→引用验证）。

Synthos 只拆成7个认知原子，因为：

```
工程粒度 vs 认知粒度：

ARC 的"筛选文献"（Stage 5）      = Synthos 的"知识获取"（原子1）的一部分
ARC 的"提取知识"（Stage 6）      = Synthos 的"知识提取"（原子2）
ARC 的"聚类主题"+"生成假设"       = Synthos 的"关联发现"+"观点生成"（原子3+4）
(Stage 7-8)
```

7个原子已经覆盖了 ARC 从文献到论文的 **全部认知环节**（18/23阶段）。
剩下的5个阶段（9-13: 实验设计和代码执行）是 ARC 独有的能力，与 Synthos 定位不同。

## 六、自我进化：持续吸收的认知生态

Synthos 的进化不依赖 Python 代码重写，而是通过：

```
外部搜索 → 评估 → [你批准] → 吸收 → 集成 → 基准测试 → 验证
                                ↓
                        skill_count++
                        skill_tree增长
                        evolution-log记录
```

从 AutoResearchClaw 已吸收：
1. **引用验证** (4层) — 嵌入原子1，消除幻觉
2. **LaTeX输出** — 新技能，扩展输出格式

待吸收：
3. **结构化知识库** (6类KB) — 让运行间知识可检索
4. **Lesson进化增强** — 让进化引擎从错误中学习
