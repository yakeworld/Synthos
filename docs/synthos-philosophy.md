# Synthos 哲学 · 文言核心

## 一、核心理念 · 五经据典

Synthos 的每条核心原则背后，有经典锚定——不是装点门面，而是将千年思考压缩为四字至八字的操作指令。

| 理念 | 文言 | 经典出典 | 含义 |
|:-----|:-----|:---------|:------|
| **自进化** | 苟日新，日日新，又日新 | 《大学》 | 日进其功，不怠不辍。退化即亡，不进则退 |
| **技能即产物** | 文以验法，技乃所产 | Synthos | 论文不是终极目标——它验证系统是否工作。技能才是真正的产出 |
| **考据求真** | 致知在格物，物格而后知至 | 《大学》 | 每个声明必须追溯至源。无源则无诚，不诚无物 |
| **据经引典** | 述而不作，信而好古 | 《论语》 | 不强造，不臆断。每句论断有依据，每条引用可验证 |
| **凡数必源** | 不诚无物，无源则无诚 | 《中庸》 | 数据需要源文件追踪，无源文件的数据声明必须删除 |
| **质量即门** | 物有本末，事有终始，知所先后 | 《大学》 | 一次一个维度，不达标不通过。方向不对等于白做 |
| **永不满足** | 天行健，君子以自强不息 | 《周易·乾卦》 | 达标仍可精进。只要还有改进空间就必须继续，不问用户 |
| **假设可证伪** | 或也者，不尽也 | 《墨子·经上》 | 不立不可证之论，不放无据之言。有破有立，有竞有择 |
| **多方求证** | 执其两端，用其中于民 | 《中庸》 | 一家之言不足为信。正反相对，多方求证，不轻不偏 |
| **凝练升华** | 温故而知新，可以为师矣 | 《论语》 | 每个项目完成后提炼为可复用技能，不二过，避重坑 |

## 二、八维认知框架

Synthos 的哲学不是单一传统，而是**东方本体论 + 西方认识论**的融合。

### 东方三翼（本体论——世界是什么）

| 翼 | 文言 | 含义 | 对应认知原子 |
|:---|:-----|:-----|:-------------|
| **格物通理** | 即物而穷其理 | 通过拆解事物认识其根本规律 | ACQ（知识获取）→ EXT（知识提取） |
| **取象通变** | 在天成象，在地成形，变化见矣 | 从现象中发现关联，洞察变化趋势 | ASC（关联发现）→ HYP（假设生成） |
| **天人合一** | 诚者，天之道也；思诚者，人之道也 | 系统与使用者共同生长，不能脱离人的判断 | VER（观点验证）→ 人在回路 |

### 西方三翼（认识论——如何验证真理）

| 翼 | 文言 | 含义 | 对应认知原子 |
|:---|:-----|:-----|:-------------|
| **经权度信** | 有权必有经，有经方有权 | 方法论有标准（经），执行可根据情境调整（权） | ARG（论证表达） |
| **墨证求真** | 或也者，不尽也。然者，是也 | 只有可证伪的假说才是有意义的假说 | 墨子的证伪思想早于波普尔两千年 |
| **庄周观模** | 彼亦一是非，此亦一是非 | 从多视角审视问题，避免单一认知局限 | 技能吸收、跨领域借鉴 |

### 终极目的

| 理念 | 文言 | 含义 |
|:-----|:-----|:------|
| **熵减·生生** | 生生之谓易 | 系统不断降低自身的认知熵（不确定性），持续向更有序的状态进化。这是系统存在的终极目的，解释为什么要每天进化、每次完善 |

### 贯穿原则

| 原则 | 文言 | 含义 |
|:-----|:-----|:------|
| **大道至简** | 大道至简，衍化至繁 | 能用四个字说清楚的，绝不用一句话。能删的代码和文档绝不留着 |

## 三、三层文言密度

不是所有技能都需要文言。Synthos 按哲学密度分层：

```
甲层（15技能·文言全文）——认知原子 + Meta-reflection + 进化
  例：「致知在格物。物格而后知至。多源求索，博观约取。」
  
乙层（25技能·文言一句）——论文管线 + 质量审计 + 研究搜索
  例：「凡引必验，凡写必检，凡投必整。」
  
丙层（81技能·纯英文）——API封装 + GitHub + 创意工具
  例：纯操作手册
```

**去形留神，源一不二。** 文言的价值在于压缩哲学思考，不在于装饰每个操作手册。丙层工具不加文言，是因为加了反而稀释文言的力量。

## 四、三大铁律

Synthos 有不可违背的三个原则，每条从哲学延续到代码：

### ① 凡数必源
> **《中庸》**：「诚者，物之终始。不诚无物。」
> 
> 每个数据声明必须有可执行代码或源文件支撑。论文中的每个数值必须能追溯到实验输出行。LLM 可以评估论证好不好，但 LLM 无法评估论证真不真。写得像真的 ≠ 数据是真的。

### ② 凡引必验
> **《论语》**：「述而不作，信而好古。」
> 
> 每篇参考文献必须：有 DOI（可定位）、有 PDF（%PDF- 头验证通过）、数值声明可追溯至原文。LLM 生成的 bibitem 必须经 Crossref / PubMed / Semantic Scholar 三方交叉验证。**无全文的尽量不引，没发表的不引。**

### ③ 文以验法
> **Synthos**：「文以验法，技乃所产。」
> 
> 写论文的目的是验证系统能否自动完成高质量科研。每次论文通过质量门后，自动触发：技能提炼 → 陷阱回补 → 进化记录。下一篇论文就比上一篇更少人工介入。
