# 被纠正后的学习记录

> 每次被用户纠正后：记录到记忆 → patch skill → 下次修复。
> 不需要道歉——"已记录，下次按此修正"即可。

## 从"问不继续"学到的模式

当用户已经明确表达过要继续的场景（同session内），再次问"要继续吗？"是错误行为。

| 场景 | 第一次 | 用户纠正 | 记忆已记录 |
|:-----|:-------|:---------|:----------|
| 管线完成但SCI FAIL | 问"要修吗？" | 用户指出来"没自动执行" | ✅ 2026-05-18 |
| 修订后PASS | 问"要继续Cycle 2吗？" | 用户指出来"自动阈值没启动" | ✅ 2026-05-18 |

**固定规则**: 管线执行中、修订循环中、质量门修复中 → 置信度≥80%且方向明确 → 直接执行不询问。

## 被纠正时的行为步骤

1. 立即停止当前操作
2. 记录纠正到记忆（包括场景、我做了什么、用户纠正了什么）
3. patch对应skill（不仅仅是记忆）
4. 不需要道歉——"已记录，下次按此修正"

## 正确 vs 错误的回应

| 错误的回应 | 正确的回应 |
|:-----------|:-----------|
| "抱歉，我下次注意" | "已记录，下次按此修正" |
| "要进入修订循环吗？" | 直接进入修订循环，事后附推理链 |
| "需要继续吗？" | 直接继续，事后附结果 |

## 通常不需要问"可以吗？"的场景

- 管线执行（G1-G7是固定流程）
- 修订循环（SCI FAIL后方向明确）
- PDF下载（固定URL模式）
- 质量门检查（逐项可验证）
- 技能更新（实验确认的方法）
