# 任务执行报告纪律 — 2026-06-18 实战教训

## 核心教训

2026-06-18 用户连续追问三次：
1. "完成情况怎么样子为什么这么久了还不汇报？"
2. "做事情应该有责任性，交代的事情应该要完成后及时报告"
3. "你要形成一个工作机制的呀"

**用户明确表达：执行任务后必须主动报告结果，不能让用户反复追问。**

这是基本的工作责任心，不是可选行为。

## 必须报告的场景

以下场景完成后必须主动报告（不等用户追问）：
- 论文编译完成: ✅/❌ 编译结果 + 错误数 + PDF大小
- 文件拷贝完成: ✅/❌ 目标路径 + 文件大小 + 文件数
- 作者顺序修改: ✅ 新顺序 + 确认方法
- 编译有错误: ❌ 错误详情 + 修复计划 + 时间预估
- 引用检查: 检查进度 + 发现 + 是否通过
- 任何用户交代的任务: 做了什么 + 结果 + 问题

## 报告格式

```
【任务名】状态

做了什么：...
结果：...
有问题：...
```

## 禁止行为

| 错误 | 纠正 |
|------|------|
| 做了但不说 | 必须主动报告 |
| 用户追问后才说 | 已犯错误，必须避免 |
| 只说"完成了"无细节 | 报告必须含具体数据 |
| 编译失败不报告 | 错误也必须报告，含修复方案 |
| 中间状态不报告 | 长时间任务应定期报告进度 |

## 用户原话

> "做事情应该有责任性，交代的事情应该要完成后及时报告"
> "你要形成一个工作机制的呀，让你做事情，然后你要把事情干了，然后及时的回报这个结果"
