# 系统描述论文的动灵(Entelechy)两阶段评估协议

> 凝练自 2026-05-23 Synthos论文两轮评审实践。适用于评审"本文描述了一个XX系统"类的论文。
> 原理：系统描述论文的"数据"不是实验数值，而是系统架构和进化轨迹。评审重点应从实验A/B对比转向系统自身的一致性和生长性。

## 触发条件

当被评审论文是系统描述类（描述一个软件/AI/架构系统而非提出新实验方法）时，在标准7维SCI评审之前，先执行L0动灵层评估。

判断依据：论文标题含"system/architecture/framework/platform/OS"，或Abstract的核心主张是"我们构建了一个XX系统"而非"我们提出了一种XX方法"。

## 两阶段协议

```
                        L0 动灵评估（方向+生化+生长+原生+自律）
                                ↓ 通过
                        标准7维SCI评审
                                ↓
                    找当前最低分维度（一次一维）
                                ↓
                    针对性修复 → 编译验证 → 重评
                                ↓
                 PASS → 完成 | FAIL → 继续下一维
```

## 阶段1: L0 动灵五维评估

### 五维度定义

| 维度 | 1-2分 | 3分 | 4-5分 |
|:-----|:------|:----|:------|
| **方向性** (Directionality) | 纯操作手册，无哲学根基 | 有哲学痕迹但浅，像贴标签 | 哲学是基因和灵魂，驱动每个决策 |
| **生化度** (Biochemical) | 原样搬运，吸收来源标记仍在 | 部分改写但残留外部痕迹 | 完全消化为原生表达，看不出来源 |
| **生长性** (Growth) | 无版本/无变更日志 | 有版本号但无实质迭代 | 版本+门控完整，每次迭代可追溯 |
| **原生感** (Native feel) | 显然从别处搬来的 | 有生长痕迹但外观不整 | 浑然天成，与系统原生部分不可区分 |
| **自律性** (Autonomy) | 强依赖外部输入/审批 | 部分依赖，有兜底但不完整 | 自洽自驱，有完整边界判定 |

### 评分规则

- 每维 1-5 分
- 平均分 >= 4.0 => 动灵健康，进入7维评审
- 平均分 3.0-3.9 => 需关注，记录薄弱维度，进入7维评审
- 平均分 < 3.0 => 需修复，先修复动灵最低维再进入7维评审
- 生化度 < 2.0 => 标记为"聚集式吸收"，需重新走五层提取流程

### 三种反动灵模式检测

在评审过程中，主动检查以下三种模式：

| 模式 | 诊断线索 | 扣分影响 |
|:-----|:---------|:---------|
| **贴标签式哲学** — 文言/哲学在开头但技能设计与哲学无关 | 去掉文言后方法层仍然自洽 | D1 科学贡献上限-0.10 |
| **聚集式吸收** — 外部内容原样堆叠，无统一形式因 | 多个"吸收自XXX"标记、Phase1->Phase2线性堆积 | D2 方法学上限-0.15 |
| **经济理性假驱动** — "省时省力"替代"完满实现"作为终极理由 | 原理层讲"减少负担"而非"系统生长" | D1/D6各-0.05 |

### 证据要求

每维必须附：
- score: 1-5
- evidence: 具体段落号、语句引用
- finding: 一句话的关键发现

## 阶段2: 标准7维SCI评审

按 sci-paper-quality-review 的7维标准执行，但特别关注系统描述论文的特殊扣分点：

| 维度 | 系统描述论文的特殊检查 |
|:-----|:---------------------|
| D1 科学贡献 | 系统与同类系统的区分度（对比表是否清晰） |
| D2 方法学 | 架构是否充分解释？形式化定义？I/O契约体现？ |
| D3 结果可信度 | 进化数据是否验证？代码开源？最易出问题：论文数据落后系统当前状态 |
| D4 完整性 | 原子数量是否全文一致？图表是否齐全？ |
| D5 清晰性 | 吸收表是data dump还是学术提炼？ |
| D6 新颖性 | "别人没做到的"是否清晰？ |
| D7 引用质量 | GitHub引用占比？自引率？ |

## 阶段3: 单维修复循环

### 优先级

系统描述论文的修复优先级（已验证：Synthos两轮修复合集）：

1. **D3/生长性（数据更新）** — 最易修复、单轮收益最大（+0.30~+0.50）
   - 更新版本号、cycle数、质量分
   - 补全新吸收记录
   - 增补最新架构变更

2. **D5/清晰性（吸收表重塑）** — 中等工作量（+0.10~+0.15）
   - 吸收表从data dump改为分类分组
   - 增加通俗类比降低哲学门槛

3. **D4/完整性（数字统一）** — 低工作量但收益显著（+0.10~+0.15）
   - 统一全文的原子/组件数量
   - 统一吸收记录的数字
   - 确保 Abstract、正文、表格数据一致

4. **D7/引用质量** — 持续优化（+0.05~+0.10）
   - 减少GitHub引用占比
   - 补充学术版本引用

### 修复铁律

| 规则 | 原因 |
|:-----|:------|
| 一次一维 | 防止多维度同时改导致的退化难以定位 |
| 每轮修复后编译验证 | 确保语法正确性 |
| 每轮修复后重新评估 | 确保改进有效而非倒退 |
| 有进展就一直循环 | 不设硬性上限 |
| 连续3轮无进展降级目标 | 避免资源浪费 |

### 编译验证

修复后的论文必须通过 pdflatex 编译：
```bash
cd <paper_dir>
pdflatex -interaction=nonstopmode paper.tex
# 检查 Output written on paper.pdf
# 检查无 ! Error 行
```

## 验证清单

- [ ] Abstract数据与正文Table一致
- [ ] 原子/组件数量全文统一
- [ ] cycles/版本号/质量分与当前系统状态匹配
- [ ] 吸收记录数量与Table一致
- [ ] 无"吸收自XXX"直提（已消化为原生表达或转为reference）
- [ ] 动灵概念有完整的叙事线（Abstract->Architecture->Results->Discussion->Limitations）
- [ ] pdflatex编译通过
