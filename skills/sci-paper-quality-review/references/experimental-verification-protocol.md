# Experimental Verification Protocol (EVP)

> 当论文声称在公开数据集上做实验时，运行实验验证声明值是否真实。
> 吸收自 2026-05-26 PIMA CRISP-DM Helix 论文 L0.5 审计实战。

## 触发条件

论文包含以下任一特征时，进入 EVP 流程：
- 声称在公开数据集（PIMA, WDBC, MNIST, CIFAR, ImageNet 等）上得到结果
- 有定量数值声明（F1, Accuracy, AUC 等）且声称来自实验
- 消融研究/对比实验/基线对比

## 三步验证流程

### Step 1: 独立实验（写干净的实验代码）

不要尝试复现论文的“精确代码”——写一份独立、干净的实验代码：

```bash
# 创建实验目录
mkdir -p experiment/results

# 关键原则：
# 1. 从标准源下载数据（UCI/OpenML/sklearn）
# 2. 按论文描述的方法实现管线
# 3. 严格 CV 隔离预处理（如果论文声称这样做）
# 4. 输出所有数值到 JSON/CSV
```

### Step 2: 数值对比表

```python
paper_claims = {
    "ensemble_f1": 0.7541,
    "ensemble_recall": 0.7500,
    "no_leakage_f1": 0.6759,
}

actual_values = {
    "ensemble_f1": 0.6986,   # 实验值
    "ensemble_recall": 0.7500,
    "no_leakage_f1": 0.6759,  # 精确匹配！
}

# 评估匹配度
# ✓ 精确匹配 (diff < 0.005): 数据真实
# ✓ 容差内 (diff < 0.02): 随机变化，可接受
# ✗ 超出容差 (diff >= 0.02): 声明值不可信
```

### Step 3: 修复决策

| 对比结果 | 操作 |
|:---------|:-----|
| 所有数值在容差内 | ✅ 论文数据通过验证 |
| 部分匹配部分不匹配 | 📝 只保留匹配值，用实验值替换不匹配值 |
| 完全不匹配 | ⚠️ 全文替换为实验值，论文来自LLM生成 |

## 关键战术发现

### 1. 基础模型 vs 集成结果的内部一致性检查

PIMA 审计的关键发现：**集成结果不能显著高于所有基础模型的最好结果。**

```text
单个模型 F1 范围: 0.6414 (MLP) ~ 0.6857 (GBC)
集成(GBC+LDA+SVC) F1: 0.6986  ← 合理（略高于最佳单模）
论文声称集成 F1: 0.7541        ← 不合理（比最佳单模高 0.07）
```

**规则**: 集成 F1 不应超过最好单模型 F1 + 0.03（软投票集成的理论上限）。如果声称超过，需要先验证单个模型结果。

### 2. 消融实验数据一致性检查

如果消融实验的趋势不合理，很可能是虚构的：

```text
正确趋势（PIMA 真实数据）:
  No Leakage:      F1=0.6986  ← 基线
  Severe Leakage:  F1=0.7657  ← 更高（泄漏虚高）
  
错误趋势（论文原声明）:
  No Leakage:      F1=0.6759  ← 基线（来自 LDA 非集成）
  Severe Leakage:  F1=0.7338  ← 更高但数值不对
```

**规则**: 消融实验中，泄漏程度越严重，F1 越高同时 Recall 越低。如果这个 trade-off 不成立，数据有问题。

### 3. LDA/简单模型作为 "无泄漏基线" 是一种混淆

PIMA 论文中"无泄漏基线 F1=0.6759" 精确匹配 LDA+SMOTE 的实验值，但论文声称这是集成模型的基线——实际上是 LDA 单模型的结果。**论文混淆了无泄漏基线应该用哪种模型。**

**规则**: 检查无泄漏基线是否与论文描述的方法一致（集成 vs 单模型）。如果基线值匹配一个简单模型但论文声称是复杂模型的结果，说明论文在混淆概念。

### 4. 阈值优化能缩小但消除不了差距

```python
# 使用最佳阈值 (thresh=0.42) 使 F1 从 0.6986 升至 0.7071
# 但仍远低于声称的 0.7541
```

**规则**: 阈值调优最多能提升 F1 约 0.01-0.02。声称 F1=0.7541 但实验最佳仅 0.7071（差距 0.047）是不可调和的。

## 已验证数据集基线（供对比参考）

### PIMA Indians Diabetes Dataset

| 模型 | F1 | Recall | 备注 |
|:-----|:--:|:------:|:-----|
| GBC+SMOTE (隔离) | 0.6857 | 0.7464 | 最佳单模 |
| LDA+SMOTE (隔离) | 0.6759 | 0.7091 | 匹配论文基线 |
| Ensemble (隔离) | 0.6986 | 0.7500 | 真实集成 |
| Ensemble (严重泄漏) | 0.7657 | 0.6364 | F1 虚高，Recall 下跌 |
| GBC (阈值优化) | 0.7071 | — | 最佳调整值 |
| GBC (基础) | 0.6529 | 0.6232 | 无 SMOTE |

**PIDD 真实性能上限**: F1 = 0.6986 (严格隔离), F1 = 0.7071 (阈值调优)
**任何声称 F1 > 0.73 的论文应自动进入方法学审计。**
