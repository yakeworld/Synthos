# 修订循环规范

> SCI 评审不通过 -> 自动（不提问）定位最弱维度 -> 修复 -> 重审。
> 最多3次循环，3次不过降级目标。
> 反模拟铁律：每次修订必须基于 read_file 读取的实际内容。

## 循环流程

```
G7通过 -> SCI评审
            ↓
        avg >= 0.80? -> ✅ PASS，报告
            ↓ (avg < 0.80)
    找7维中最低分维度
            ↓
    read_file("paper.tex") 确认当前内容
            ↓
    分析缺口(什么缺/为什么缺/怎么补) 
            ↓
    执行修复(只改最低分维度，不碰其他)
            ↓
        重审
            ↓
        PASS? -> ✅ 报告
            ↓
        FAIL? -> 第2轮... 第3轮...
            ↓
        3次FAIL? -> 降级: 标记为非SCI标准
```

## 一轮只修一个维度

| 错误做法 | 正确做法 |
|:---------|:---------|
| 发现D3=0.55, D4=0.65 -> 同时改两个 | 选最低的D3(0.55)，只修这一个 |
| 改D3时顺手修了排版 | D3改完->重审->通过->新的SCI评审 |
| 凭记忆估计修复效果 | read_file 确认修复内容 |

## 反模拟检查点

每次修订循环开始前，回答以下问题：

- [ ] 最低分维度是哪个？（来自 SCI 评审结果）
- [ ] 该维度的当前分数证据是否来自 read_file？
- [ ] 修复方案是否基于实际论文内容（不是"增加一段"而是具体写什么）？
- [ ] 修复后：重审是否基于 read_file 读取修改后的版本？
