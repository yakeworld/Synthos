# 民进提案格式规范

> 基于民进INSTITUTION_NAME_PLACEHOLDER支部两份现有提案（论文晋升提案+基层创新建议）的格式分析提炼。
> 2026-05-26 人才Token提案实战中用户纠正后确认。

---

## 一、结构规范

### 1. 标题
- 格式：`关于……的提案`
- 纯文字，不加粗，不居中
- 示例：
  ```
  关于取消论文作为医务人员初级晋升中级必要指标的提案
  ```

### 2. 提案人信息
- 格式：`提案人：XXX  民进XX支部/XX单位`
- 位于标题正下方
- 示例：
  ```
  提案人：杨晓凯  民进INSTITUTION_NAME_PLACEHOLDER支部
  ```

### 3. 正文结构（三段式）

```
1.基本情况                    ← "是什么"
  直接陈述现状、背景、数据     ← 不分小节，段落展开

2.存在问题及其原因分析         ← "为什么"
  表象→直接原因→根本原因       ← 逻辑递进，不分小节

3.对策建议                    ← "怎么办"
  具体、可操作的政策建议       ← 每条建议段落式表述
```

**铁律**：三段之内不使用任何次级编号（如（一）、1.、①等），全部以段落自然展开。

---

## 二、格式约束

### 1. 编号
- 一级标题使用：`1.`、`2.`、`3.`（阿拉伯数字+实心点）
- 标题与正文之间无空行
- 示例：
  ```
  1.基本情况
  正文内容...

  2.存在问题及其原因分析
  正文内容...

  3.对策建议
  正文内容...
  ```

### 2. 段落
- 不分小节、不列小标题
- 每个逻辑层使用2-4个段落展开
- 段落之间空一行
- 不缩进

### 3. 排版禁令
| 元素 | 禁止/允许 |
|:-----|:---------:|
| 表格 | ❌ 禁止 |
| 加粗 | ❌ 禁止 |
| 斜体 | ❌ 禁止 |
| 项目符号（•、-、*） | ❌ 禁止 |
| 编号列表（1.、a.等正文内） | ❌ 禁止 |
| 流程图/思维导图 | ❌ 禁止 |
| 纯文字段落 | ✅ 必须 |

---

## 三、内容规范

### 1. 数据引用
**嵌入正文式引用**：将数据来源直接写入句子，不加脚注/尾注/参考文献列表。

✅ 正确：
```
根据CNKI数据库2020年发表论文检索结果，每年刊发期刊论文约290万篇……
国家卫生健康委统计公报显示2019年末全国卫生人员总数达1292.8万人……
人力资源社会保障部和国家卫生健康委共同研究起草了《关于深化卫生专业技术人员职称制度改革的指导意见（征求意见稿）》……
```

❌ 错误：
```
全国医师发文需求巨大[1]。代写代投现象严重[2]。
[1] CNKI, 2020
[2] 卫健委公报, 2019
```

### 2. 语言风格
- **直入主题**：第一句即进入事实，不寒暄铺垫
- **数据驱动**：每个论点尽量有数据支撑
- **一事一议**：一份提案只聚焦一个问题，不旁逸
- **客观克制**：不夸张、不煽情、不评价、不指责
- **精准量化**：用数字说话，避免"大量""很多""严重"等模糊表述
- **建议可操作**：每条建议明确"谁来做、怎么做、何时做"

### 3. 篇幅控制
- 总字数：800-1500字
- 每段：3-8行
- 三段比例：基本情况:问题分析:对策建议 ≈ 1:1.5:1

---

## 四、注意事项

### 1. 建议要"可操作"
- 每个建议要明确"谁来做、怎么做"
- 避免笼统表述，如"加强管理""提高重视"

### 2. 结论要"可采纳"
- 建议应在接收单位的职权范围内
- 不提跨层级、跨部门的宏大方案

### 3. 立场要"建设性"
- 问题可以尖锐，但态度是"帮忙解决问题"而非"批评指责"
- 分析原因要客观，指向制度而非个人

---

## 五、提交前检查清单

```
□ 标题为"关于……的提案"格式
□ 提案人信息完整（姓名+支部）
□ 三段结构完整（基本情况→问题分析→对策建议）
□ 无次级编号（无（一）、1.等）
□ 无表格
□ 无加粗/斜体文字
□ 无项目符号/编号列表
□ 数据引用嵌入正文
□ 一事一议
□ 建议可操作（有责任主体+实施路径）
□ 总字数800-1500字
□ 立场建设性
```
