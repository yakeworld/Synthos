# Synthos 吸收标准体系

> 外为养，内为化。不搬不抄，取法乎上。

---

## Part 1: Synthos 思想体系（动灵哲学）

Synthos 是一套自进化的认知操作系统，其思想由 8 条核心原则构成：

### P1: 动灵在内，不假外求

> 系统的生长力量在内不在外。外部项目是养料，不是器官移植。
> 吸收不是"缺什么补什么"，而是"看别人怎么做的，内化成自己的方法"。

**判断标准**：吸收后，外部项目的形态消失，其方法论以 Synthos 原生结构存在。

### P2: 短悍精小

> 技能非越长越好，非越全越好。割掉冗余，留下精华。
> 一条技能只做一件事，做到极致。超过 300 行必有冗余。

**判断标准**：核心技能 < 300 行，参考技能 < 250 行。每条技能聚焦一个认知操作。

### P3: 文言为魂，白话为体

> 核心哲学用文言表达——四字格言，一字不可易。
> 具体操作用白话说明——清晰准确，不啰嗦不模糊。

**判断标准**：每条技能必有 ≥3 条四字文言格言。文言在白话之前。

### P4: 凡法必源，凡数必验

> 每个方法论声明必须有来源追溯。每个数字声明必须可独立验证。
> 不虚构、不编造、不臆测。宁无所得，不取伪术。

**判断标准**：无虚构数据，无不可追溯的声明。所有来源标记 provenance。

### P5: 域有其模，模载其法

> 每个领域有自己的认知模板。加新域=加新模板，不改核心。
> 模板是方法论的可移植载体。

**判断标准**：核心原子不随领域变化。新领域通过模板扩展。

### P6: 一维一修，一步一验

> 每轮只修一个维度。每次修改前写假说。每次假说必有可证伪预期。
> 先立说后动刀，证不成则返。败亦有训，不徒弃之。

**判断标准**：IMPROVE 前必输出 hypothesis_preamble。VERIFY 有四态决策。

### P7: 拒绝搬运，只取方法论

> 不把外部项目的代码/SKILL.md 直接复制进来。
> 读其思想，取其方法论，用 Synthos 形态重新表达。

**判断标准**：无 "preserved as-is from" 标记。无外部引用行。文言提炼在前。

### P8: 碳硅共生，人为决策

> 人在回路做决策，AI 执行任务。系统越强，人的杠杆越大。
> 人定方向，AI 写论文。人审质量，AI 迭代。

**判断标准**：关键决策点产生人类可读选项。不替代人的学术判断。

## Part 2: Synthos 技能标准

### 2.1 结构模板

```
---
[YAML frontmatter]
name:           技能名称（英文小写）
description:    一句话描述（<200字）
version:        semver
author:         Synthos
license:        MIT
allowed-tools:  允许的工具列表
metadata:
  synthos_atom_type: cognitive | extended | reference | pipeline
  synthos_data_access_level: raw | redacted | verified_only
---

# 技能名称（中文）

## 原理层·文言

> [3-5 条文言四字格言]
> [一句总结]

## 方法层·白话

[一句话说明这个技能做什么]

## 触发条件

[何时加载此技能]

## [核心方法论]

[技能特有的方法论内容]

---

## 验证清单

- [ ] 执行前检查项
- [ ] 执行后检查项

## 命令层（可选）

[工具的签名/契约信息]
```

### 2.2 质量门

| 门 | 标准 | 不通过处理 |
|:---|:-----|:-----------|
| **G1 行数** | 核心≤300行，扩展≤250行，参考≤200行 | 压缩或拆分 |
| **G2 文言** | ≥3 条独立文言格言 | 注入原理层 |
| **G3 结构** | 含原理层·文言 + 方法层·白话 + 触发条件 + 验证清单 | 补充缺失段 |
| **G4 清理** | 无 "absorbed_from" / "preserved as-is" / 外部搬运标记 | 移除+方法论提取 |
| **G5 接口** | allowed-tools 与实际使用一致 | 修正 |

### 2.3 评分公式

```
结构分 = G1(0.20) + G2(0.25) + G3(0.25) + G4(0.20) + G5(0.10)

Grade:
  0.90-1.00: 文质兼美 ✅   — 完全达标
  0.70-0.89: 达标 🟡      — 可接受，小修
  0.50-0.69: 需重构 ⚠️    — 需一个修订周期
  <0.50:     不合格 ❌     — 大幅重写
```

## Part 3: 外部项目比较框架

### 3.1 五维比较

| 维度 | 权重 | 问什么 | 评分方法 |
|:-----|:----:|:-------|:---------|
| **思想相容** | 0.25 | 该项目的哲学与 Synthos 8 原则重合多少？是否有冲突？ | 逐条对比 8 原则 |
| **技能形式** | 0.20 | 该项目是否用类似 SKILL.md 的结构？还是代码/脚本/文档？ | 对照技能标准 G1-G5 |
| **集成成本** | 0.20 | 需要多大改造才能融入 Synthos 体系？ | 从方法论提取到文言注入的工作量 |
| **互补价值** | 0.25 | 填补了什么 Synthos 没有的功能/方法论缺口？ | 对比现有技能树 |
| **许可证** | 0.10 | MIT/Apache/BSD 优先？NOASSERTION? | 法律合规检查 |

### 3.2 吸收决策矩阵

| 综合分 | 决策 | 动作 |
|:------:|:-----|:------|
| ≥4.0 | 🔥 强烈推荐吸收 | 提取方法论→注入现有技能→文言提炼 |
| 3.0-3.9 | 👍 建议吸收 | 写入待办，下次进化周期执行 |
| 2.0-2.9 | ⏭️ 短期搁置 | 记录原因，30轮后自动清理 |
| <2.0 | ❌ 不吸收 | 归档，记录不吸原因 |

### 3.3 吸收流程

```
外部项目
   │
   ├── ① 读文档/代码 → 理解核心方法论
   ├── ② 五维评分    → 决定是否吸收
   ├── ③ 方法论提取  → 剥离具体实现，提取可移植原理
   ├── ④ 文言提炼    → 压缩为 3-5 条文言格言
   ├── ⑤ 注入融合    → 进入 Synthos 现有技能协议
   └── ⑥ 记录        → evolution-log 记录吸收过程
```
