# BOUNDARY.md — task-router

> 对应原则：P2

## 边界陈述

路由器的唯一职责：解析用户查询，判定复杂度，选择执行模式（简单链/内循环/双循环），输出最短执行路径。不执行任何认知操作。

## 与其他原子的边界

路由器与所有原子是调用者-被调用者关系，不存在功能重叠。

## 双循环编排边界（v1.2.0新增）

### 路由器负责
- 判断是否启用内循环/双循环模式
- 编排循环执行顺序（何时进入内循环、何时跳转到外循环）
- 跟踪循环状态（迭代次数、结果累积、退出条件）
- 记录循环元数据到 pipeline_trace.json

### 路由器不负责
- 不执行认知操作（分析/综合/验证等被调原子的专属功能）
- 不做循环内的决策内容判断（由具体原子自行输出判断依据）
- 不修改原子输出（仅传递和汇总）

## 循环编排 vs 原子认知的边界

| 操作 | 归属 | 理由 |
|------|------|------|
| 决定"进入循环还是退出" | ROUTE | 编排决策，非认知操作 |
| 判断"实验A的结果是否支持假设" | HYP/VER | 认知操作 |
| 选择"下个内循环迭代做什么" | HYP（从循环已累积结果） | 内循环中HYP原子的输入依赖累积结果 |
| 控制"每5轮触发外循环检查" | ROUTE | 定时编排 |
| 决定"继续、转向还是收尾" | 外循环由HYP+ASC综合判断，ROUTE执行决策 | ROUTE执行但不判断
