# 匿名网络项目对比

> 类 Tor 匿名网络项目的成熟度、架构差异与适用场景。

## 成熟项目

### I2P（Invisible Internet Project）
- 起步：2003年，已运行 >20 年
- 节点：~2000-3000 活跃路由器
- 架构：暗网导向（anycast），所有流量归内部，无出口节点
- 匿名性：某些场景比 Tor 更强（无出口监控风险）
- 适合：匿名服务、文件共享、匿名论坛
- 不适合：需要访问外网的场景

### Freenet
- 起步：2000年（比 Tor 更早）
- 架构：去中心化分布式存储 + Kademlia DHT
- 特点：文件分块加密分散存储，不可删除性
- 适合：不可抵赖的匿名信息发布

### ZeroNet
- 起步：2015年
- 架构：BitTorrent + Bitcoin P2PK + Tor 三合一
- 节点：数千活跃站点，社区较小
- 适合：抗审查静态网站

### GNUnet
- 起步：2002年，GNU 项目
- 架构：通用安全 P2P 网络框架，内置多种应用
- 特点：前向安全 Gossip 协议，安全性优先于性能
- 适合：研究和开发平台

### Nym（前 MixedNet）
- 起步：2019年
- 架构：从 onion 路由转向 Mixnet，真实部署的混合节点
- 特点：Proof-of-Stake 激励混合节点运营商
- 状态：建设中，代表下一代匿名网络路径

## 学术原型

- **Loopix**（2017，爱丁堡大学）：低延迟+高匿名性，填充流量+接收队列机制
- **Riffle**：双翻 onion 路由，理论安全性更强
- **Sphinx**：低延迟匿名消息传递

## 选型决策树

1. 需要访问外网 → Tor（无成熟替代）
2. 需要内部匿名服务 → I2P
3. 需要不可抵赖发布 → Freenet
4. 需要去中心化网站 → ZeroNet
5. 需要研究和协议开发 → GNUnet
6. 需要下一代架构探索 → Nym
