## IO_CONTRACT

- **input**: `template_path: str, project: dict, competition_info: dict`
- **output**: `docx_path: str` — 填写完成的申报书

## 触发条件

- 用户发送竞赛申报书模板（DOCX/PDF），需要评估是否申报
- 用户需要撰写竞赛申报书正文内容
- 用户需要将技术项目包装为竞赛评委易理解的叙事
- 用户提交的申报书需要赛道匹配性审查

## 工作流程

### Step 1: 赛道匹配性评估

**核心原则：不要看"这个项目我能报哪个赛道"——要看"哪个赛道的评审标准最有利于这个项目"**

评估矩阵：

| 赛道维度 | 评审看重什么 | 自评问什么 |
|:---------|:------------|:----------|
| 数据相关要求 | 数据量、营收、Token消耗、融资阶段 | 你的单位有这些数据吗？没有→这个赛道不适合 |
| 技术深度 | 核心技术突破、知识产权 | 你的AI方法论属于赛道底层技术还是应用层？ |
| 行业属性 | 具体行业痛点、落地案例 | 你的临床案例属于哪个行业分类？ |
| 单位属性 | 企业/事业单位匹配 | 表格问企业规模/营收→事业单位填"不适用"靠谱吗？ |

**关键检查项**：
1. 模板中是否有"营收/利润/融资阶段/企业规模"等必填字段 → 事业单位需确认能否跳过
2. 是否有"Token消耗量/数据产品交易量"等基础设施特有字段 → 实验室项目需确认匹配度
3. 评审标准是否侧重"技术创新"还是"产业规模化" → 选前者的赛道

### Step 2: 叙事策略转换

**技术语言 → 竞赛语言翻译映射表：**

| 技术语言 | 竞赛语言 |
|:---------|:--------|
| "AI Agent 自主进化" | "基于自主智能体的数据要素全生命周期治理框架" |
| "G1-G7质量门控" | "首创七级数据质量门控体系，确保实验可追溯" |
| "30模型基准测试" | "覆盖30个主流模型的系统性基准验证" |
| "数据泄露检测" | "解决AI医学论文70%可复现危机" |
| "方法论级解决方案" | "可审计、可复现的数据治理范式" |

**标准叙事结构（4段式）**：

1. **痛点** — 行业面临什么数据/技术/质量危机（用具体数据说话）
2. **方案** — 技术如何系统性解决（强调方法论而非具体模型）
3. **成效** — 已落地的具体案例和量化指标
4. **复制** — 为什么可以推广到其他领域/机构

### Step 3: 模板表格分析

**处理DOCX模板表格的要点**：

1. **识别合并单元格**：政府模板大量使用合并单元格，20列宽表格中实际只需要填少数几个cell
2. **找到空单元格**：标签列（Col 0-1）+ 填充区域（Col 2+）
3. **处理选项中选**：赛道/场景等选项通过文本标记（"√ 医疗健康"）而非修改checkbox
4. **财务字段**：医院/实验室填"0"或"不适用"，写注释说明

**常见表格坑**：
- Row 34 可能不存在（实际行数 < 预期行数）
- 合并单元格写Col 2的内容可能被相邻格覆盖
- 长文本需要手动换行保证DOCX显示正常

### Step 4: 填充实现

用python-docx程序化填充，避免手动编辑。分离内容与代码：将正文内容写入JSON文件，脚本从JSON加载后填充DOCX。

```python
from docx import Document
def set_cell(table, row_idx, col_idx, text):
    cell = table.rows[row_idx].cells[col_idx]
    for p in cell.paragraphs:
        for r in p.runs:
            r.text = ''
    p = cell.paragraphs[0]
    p.text = text
    for run in p.runs:
        run.font.size = Pt(10.5)
        run.font.name = '宋体'
        run._element.rPr.rFonts.set(qn('w:eastAsia'), '宋体')
```

### Step 5: 清理模板说明文字

填充后，模板中的说明文字（如"（介绍参赛项目的背景...）"）会残留为独立段落。需要批量识别并清空：
- 特征：**长度 > 40字符** 且包含模板占位符标记
- 通过段落索引定位，清空run内容

## 典型陷阱

### 🔴 赛道错选
**问题**：数据基础设施赛道要求营收/Token/融资数据，医院/实验室无法满足
**解决**：先排查模板中所有带星号的必填字段，确认单位类型能否跳过

### 🔴 叙事过度技术化
**问题**：用"AIAgent"、"G1-G7门控"等术语直接出现在申报书中，评审看不懂
**解决**：按照叙事策略转换表翻译，始终问"这句话对非技术背景的评审意味着什么"

### 🟡 财务字段应付
**问题**：医院全填"0"显得项目没价值
**解决**：能填的填具体数字（数据量、增速、开发成本），不能填的写"不适用（事业单位）"
