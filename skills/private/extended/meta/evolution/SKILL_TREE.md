# Synthos 技能树增长指南

## 概述

Synthos 的技能树不是静态的 — 它应该随时间增长和优化。
本指南定义技能树的组成、增长方式和质量管理。

## 技能树分层

```
技能树层次:
├── [核心层] 6认知原子 + 1路由器（7个SKILL.md）
│   └── 稳定：不修改，除非用户批准
├── [扩展层] 新技能（吸收的外部技能、定制工具）
│   └── 动态：可增长，用户批准后加入
└── [元层] 进化引擎 + 基准测试
    └── 自我优化：Agent驱动
```

## 技能树覆盖评分

评估标准（每轮进化计算）：

| 评估项 | 权重 | 计算方法 |
|--------|------|---------|
| 核心原子完整性 | 0.40 | 7个原子全部结构分≥0.7 → 满分 |
| 功能覆盖度 | 0.30 | 基准测试通过率 |
| 技能多样度 | 0.15 | 拥有的skills/目录下SKILL.md数量（基准7，每增1个+0.02） |
| 外部吸收潜力 | 0.15 | 本次进化发现的吸收候选数 × 0.05（上限0.15） |

## 增长触发条件

只有在以下条件下才建议技能树增长：
1. 基准测试发现持续失败（同一测试连续3轮不通过）
2. 外部搜索发现高度互补的项目（吸收分≥4.0）
3. 用户明确要求添加新功能
4. 八维框架中某个维度持续<50分

## 增长类型

| 类型 | 说明 | 自动/手动 | 例子 |
|------|------|-----------|------|
| 新增搜索源 | 在knowledge-acquisition SKILL.md中添加API | 手动（核心原子） | 添加bioRxiv源 |
| 新增reference文件 | 补充缺失的文档 | 自动 | 缺失EVIDENCE_SCHEMA.md |
| 扩展基准测试 | 添加新的测试场景 | 自动 | 新增ACQ-03 |
| 吸收外部技能 | 从GitHub/Hermes吸收 | 手动（需审批） | 添加文献监控能力 |
| 创建新技能 | 填补功能空白 | 手动（需审批） | 创建competition-packager |

## 技能树快照

每次进化记录 `skill_count`、`skills/` 目录内容、8维评分的演化轨迹。
存储在 evolution-state.json 的 `skill_tree` 字段：

```json
{
  "skill_tree": {
    "total_skills": 7,
    "core_atoms": 7,
    "extended_skills": 0,
    "external_absorptions": 0,
    "benchmark_pass_rate": 0.85
  }
}
```
