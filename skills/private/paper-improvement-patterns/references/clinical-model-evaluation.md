# 临床模型评估模式

## 核心原则

### 1. 筛查 vs 诊断模型区分

评估临床ML模型时，必须先明确使用场景：

| 维度 | 筛查模型 | 诊断模型 |
|:-----|:---------|:---------|
| 目标人群 | 一般/高风险人群 | 有症状患者 |
| 首要目标 | **最小化漏诊(FN)** | **最小化误诊(FP)** |
| 优先指标 | **召回率(Recall)** | **精确率(Precision)** |
| 阈值策略 | 降阈值（更敏感） | 升阈值（更特异） |
| 误诊代价 | 复查（低成本） | 不必要治疗（高成本） |
| 漏诊代价 | 延迟治疗（高风险） | — |

**糖尿病预测 → 筛查任务** → 召回率优先，阈值应降低。

### 2. 错误互补性分析

评估集成模型时，不能只看F1：

```python
# 分析流程
1. 收集各模型在所有fold上的预测（per-sample）
2. 计算每模型的 FN/FP 计数
3. 检查错误重叠：
   - CatBoost独有FN → GBC能挽救?
   - CatBoost独有FP → LR能纠正?
4. 构建针对性集成（非简单概率平均）
```

**关键指标：**
- 共同FN / 独有FN — 评估模型多样性
- 共同FP / 独有FP — 评估FP互补性
- 集成挽救数 vs 集成新增数 — 净临床获益

### 3. 阈值优化

默认阈值0.5假设FN/FP代价相等 → 筛查中不成立：

```
阈值越低 → FN↓(漏诊↓) FP↑(误诊↑) → 临床获益(筛查)
阈值越高 → FN↑(漏诊↑) FP↓(误诊↓) → 临床获益(诊断)
```

推荐阈值范围：0.35-0.48（筛查），0.50-0.65（诊断）

### 4. 跨数据集模型选择

最优模型因数据特征而异：

| 数据特征 | 适合模型 | 原因 |
|:---------|:---------|:-----|
| 小样本+连续特征 (PIDD, n≈768, d=8) | CatBoost/GBC | 正则化强，抗过拟合 |
| 小样本+分类特征 (Early Diabetes, n≈520, d=16) | CatBoost | 原生处理分类特征 |
| 大样本+低信噪比 (CDC BRFSS, n≈50K, d=22) | LR/LDA | 简单模型更鲁棒 |

### 5. 凡数必源强制流程

每个论文数值必须追溯至：

```
paper.tex 中的数值
  → all_results.json 中的记录
    → 统一实验脚本/Notebook的输出
      → 原始数据 + random seed + 确定性配置
```

**检查清单：**
- [ ] 所有数值有代码输出支撑
- [ ] JSON包含fold-level detail（非仅均值）
- [ ] 自动验证Cell比对JSON vs paper
- [ ] EDA可视化由代码生成
- [ ] 库版本号已注明

## 典型工作流

1. 加载统一实验脚本/Notebook
2. 运行Run All产生all_results.json
3. 检查FN/FP分布（各模型+集成）
4. 做错误互补性分析
5. 做阈值优化（筛查场景降阈值）
6. 跨数据集对比
7. 更新paper.tex中所有数值
8. 验证Cell确认自洽
9. 质量报告含科学性/创新性/可行性三要素
