# 数据要素×大赛 申报指南

## 竞赛概述

"数据要素×"大赛（国家数据局主办）是围绕《"数据要素×"三年行动计划》设立的全国性竞赛，设多个赛道。浙江分赛使用统一的DOCX申报书模板。

## 赛道选择

| 赛道 | 适合主体 | 避坑提示 |
|:-----|:--------|:--------|
| **医疗健康** ✅ | 医院、科研院所、医学院 | 关注临床痛点、数据赋能诊疗、科研数据治理 |
| 数据基础设施 ❌ | 数据交易平台、云服务商、AI基础设施企业 | **医院/事业单位不适用** — 要求填营收/利润/Token消耗量/融资阶段，这些数据公立医院无法提供 |
| 科技创新 | 高校、科研机构 | 适用于通用AI技术而非具体临床场景 |
| 工业制造等 | 企业 | — |

### 医疗健康赛道核心填报方向

面向评审（卫健委+数据局专家），项目包装应强调：
1. **数据要素×临床诊疗** — 多模态数据融合赋能诊断
2. **数据要素×科研诚信** — 可审计方法论解决可复现危机
3. **数据要素×普惠医疗** — 方法论可复制到基层医疗

## 申报书模板（DOCX）操作要点

### 表格结构

申报书主表采用20列复杂合并单元格结构（python-docx解析为~34行）。关键字段位置：

| 字段 | 表格行 | 列 | 说明 |
|:----|:-----:|:--:|:----|
| 项目名称 | Row 1 | Col 2 | — |
| 赛道选择 | Row 2 | Col 0-19 | 选项列表跨所有列，需手动标记选中项 |
| 项目简述 | Row 4 | Col 2 | 1000字以内 |
| 覆盖场景 | Row 5-7 | Col 2 | 多选：医疗健康、公共服务等 |
| 数据要素市场化 | Row 8 | Col 2 | 服务对象+数据来源+数据类型 |
| 数据要素价值化 | Row 9 | Col 2 | 客户数量+案例+成效 |
| 团队名称 | Row 12 | Col 2 | — |
| 团队成员 | Row 14 | Col 2 | **注意**：模板只给1行数据行（Row14），且列2-19合并为一个大cell。只能输入"姓名 | 证件类型 | 证件号码 | 单位 | 职务 | 手机 | 角色 | 是否联络人"格式的文本块。 |
| 单位名称 | Row 17 | Col 2 | — |
| 联系人 | Row 21 | Col 2 | 同样为合并单元格 |
| 单位简介 | Row 22 | Col 2 | ≤300字 |
| 核心能力 | Row 23 | Col 2 | ≤600字 |
| 财务数据 | Row 24-30 | Col 2/8/14 | 医院填"0"或"不适用" |
| 数据开发利用 | Row 31 | Col 2 | 数据集数量+治理能力+算力来源 |
| 上市融资 | Row 32-33 | Col 2 | 填"否"/"0" |
| 附件-专利 | Table 3 | — | 专利和软著表，Synthos零专利则填0 |

**注意**：段落31、33等是内容段落（替换了原标题），其后有模板说明文字段落（32、34等）需清空。

### 合并单元格陷阱

DOCX模板有大量合并单元格（特别是Row 2赛道选项和Row 14团队成员）。写入数据时：
- 对合并单元格，`set_cell(table, row, col, text)` 写入后，文本会传播到所有合并的col中
- 修复：对后续合并列执行 `for r in cell.paragraphs: for r2 in r.runs: r2.text = ''` 清空传播文本
- 或直接利用合并特性，在一个cell中写入完整文本块
