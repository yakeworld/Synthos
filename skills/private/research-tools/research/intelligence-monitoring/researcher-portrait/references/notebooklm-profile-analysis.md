# NotebookLM 研究者面貌分析 — 多轮提问工作流

> 2026-05-30 实战：对杨晓凯研究者面貌进行NotebookLM交叉验证

## 工作流

### Step 1: 创建Notebook
```bash
notebooklm create "研究者面貌分析-{姓名}"
notebooklm use <project_id>
notebooklm source add "$(cat 研究者面貌_姓名.md)" --type text --title "研究者面貌_姓名" --timeout 120
```

### Step 2: 第一轮 — 全景扫描
提问：请从5个维度全面分析（完整性/准确性/亮点/格式/缺失）

典型发现：
- 🔴 研究聚焦领域章节为空
- 🔴 论文数量标题"20+篇"与正文37篇不匹配
- 🟡 专利分类可能有错位（排序=1但标注为非第一发明人）
- 🟡 缺奖项/会议/H-index板块
- 🟢 亮点：省自然基金3项、已授权发明5项、量产产品

### Step 3: 第二轮 — 深层挖掘
提问：基于第一轮发现，具体建议补充（研究领域/H-index/合作网络/数据矛盾）

典型发现：
- 研究聚焦领域可总结为4个方向（BPPV/前庭功能/神经变性/AI）
- H-index估算8-15，总引用300-800次
- 核心合作网络：中科院苏州医工所、上海大学、温州医科大学
- 科研项目存在1项重复（ChatGPT教改项目同时出现在科研和教改两个表格）
- 论文标题建议改为"40+篇"

### Step 4: 修正闭环
根据NotebookLM分析结果逐项修正：
1. 补全9个研究聚焦领域描述
2. 修正论文标题数字
3. 调整专利分类
4. 新增"待补充信息"章节

## 注意事项

| 事项 | 说明 |
|:-----|:------|
| 上传前剥离YAML | 如有YAML frontmatter需先剥离再`$(cat)` |
| 多轮串行 | 不要合并问题，一轮一个维度 |
| 评分警惕 | NotebookLM评分可能偏高+0.05-0.15，视为上限 |
| 标注出处 | Gemini的回答需要附来源引用标记 |
