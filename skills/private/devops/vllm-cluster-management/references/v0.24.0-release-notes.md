# vLLM v0.24.0 Release Notes — Key Points for Deployment

**Source**: https://github.com/vllm-project/vllm/releases/tag/v0.24.0
**Date**: 2026-06-30
**Status**: ✅ **STABLE RELEASE** (not pre-release, published 2026-06-29)
**Contributors**: 571 commits / 256 contributors (77 new). Largest release in vLLM history.

> **⚠️ Note**: This was previously misidentified as a pre-release. It is a full stable release.

## Breaking Changes (⚠️ Critical for Deployment)

### 1. CUDA_VISIBLE_DEVICES Deprecation
- vLLM no longer sets CUDA_VISIBLE_DEVICES internally
- New `device_ids` argument provided instead
- ROCm: deprecation window for CUDA_VISIBLE_DEVICES begun
- **Impact**: Any custom Docker startup scripts or systemd units that depend on CUDA_VISIBLE_DEVICES being set will break

### 2. PyTorch Version Upgrade
- Torch 2.11 on ROCm
- torch-xpu 2.12
- Torch Stable ABI migration completed for all kernels (MoE, Marlin, Machete, final _C library)

### 3. v0.23/v0.24 General Deprecations
- First-generation Qwen and QwenVL models deprecated
- Transformers v4 support deprecated

## Major Architecture Changes

### MRv2 (Model Runner V2)
- Quantized models supported by default
- GraniteMoE enabled by default
- Qwen + DeepSeek-V2 MoE models migrated
- DFlash speculative decoding support
- More accurate FP32 Gumbel sampling

### Streaming Parser Engine (New)
- Unified tool-call/reasoning parsing across models
- Parsers for: Qwen3, MiniMax-M2, GLM-4.7/5.1/5.2, Nemotron V3

### Rust Frontend Matures
- API-key authentication
- CORS support
- New endpoints: /tokenize, /detokenize, /pause, /resume, /is_paused, /abort_requests, /get_world_size, thinking_token_budget
- Python bridge for Rust tool parsers

## New Quantization Methods
- Online FP8 per-token-per-channel (PTPC) quantization
- FlashInfer cutedsl NVFP4 GEMM backend
- cutedsl MXFP8 linear kernel
- GGUF quantization migrated to plugin
- AWQ format on Intel XPU

## New Model Support
- MiniMax-M3 (full support + BF16/FP8 indexer, MXFP4, FP8 sparse GQA)
- DiffusionGemma (diffusion LLM, CPU path)
- Hierarchical Reasoning Model, OpenMOSS
- Llama4, Gemma 4, Qwen3.5, DeepSeek-V4
- Cohere2 MoE, Nemotron V3
- ViT full CUDA Graph: GLM-4.1V, DeepSeek-OCR, Kimi-VL, mllama4, Lfm2VL

## Hardware Support Updates
- **NVIDIA**: SM90/SM100/SM120 support, CUTLASS FP8 mm (180-290% kernel speedup), native DSA indexer on SM100
- **AMD ROCm**: Torch 2.11, fused AR+RMSNorm+FP8, DSv4 flash-decode, AITER attention backend
- **Intel XPU**: Sequence-parallel, W4A16 int4 MoE
- **CPU**: 2.5x ASR preprocessing, W4A16 INT4 MoE, cgroup memory-aware KV cache
- **TPU**: tpu-inference v0.22.1

## Security Hardening
- DoS protection: audio decompression bomb, speculative decoding token reinjection, prompt_embeds on M-RoPE
- Info leak: int32 truncation in GGUF dequantize kernels
- Starlette upgraded to ≥ 1.0.1 (fix CVE-2026-48710)

## Removed Models
ERNIE, Xverse, Dots1, Bamba, Mono-InternVL, InternLM registry alias

## Speculative Decoding
- EAGLE3 support for Qwen3
- Dynamic SD (speculative decoding)
- DFlash + FlashInfer
- Mixed KV page sizes

## Large-Scale Serving
- DeepEP v2 for expert parallelism
- NIXL EP (elastic expert parallel communicator)
- KV connectors: NIXL push from prefill to decode
- KV offloading: multi-tier async batched lookup, packed HMA layout
