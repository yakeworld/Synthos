# Three-Dataset Leakage Pattern: PIDD vs Heart vs WDBC

> 实战来源: crispdm-wdbc 全流程 (2026-06-04)
> 核心发现: 数据泄漏的伤害模式不是单一的——存在三种不同泄漏效应模式

## 三数据集对比表

| 数据集 | N×F | 患病率 | 基线F1 | 泄漏F1 | ΔF1 | ΔRecall | 模式 |
|:-------|:---:|:------:|:------:|:------:|:---:|:-------:|:-----|
| **PIDD** (Pima) | 768×8 | 34.9% | 0.6986 | 0.7657 | **+9.6%** | **−11.4%** | Recall悖论 |
| **Heart** (Cleveland) | 303×13 | 45.9% | 0.7886 | 0.9004 | **+14.2%** | **+10.2%** | 均匀膨胀 |
| **WDBC** (Breast) | 569×30 | 37.3% | 0.9693 | 0.9683 | **−0.1%** | **+0.6%** | 可忽略 |

## 三种泄漏模式

### 模式A: Recall悖论 (PIDD)
```
F1  ↑ +9.6%
Rec ↓ -11.4%
→ 最危险的模式：指标看起来更好，实际临床敏感度下降
→ 触发条件：基线模型在决策边界附近 + 重度SMOTE过采样
→ 检测：同时在论文中报 F1 和 Recall，看方向是否相反
```

### 模式B: 均匀膨胀 (Heart)
```
F1  ↑ +14.2%
Rec ↑ +10.2%
→ 迷惑性但非破坏性：所有指标统一膨胀
→ 触发条件：数据集有足够提升空间 + 漏泄充足
→ 检测：指标方向一致但绝对值虚高
```

### 模式C: 可忽略 (WDBC)
```
F1  ↓ -0.1%
Rec ↑ +0.6%
→ 泄漏在高分离度数据集上无实际影响
→ 触发条件：基线F1>0.96，决策边界已近最优
→ 检测：标准CV提供足够保护，无需严格隔离
```

## 对论文写作的影响

| 场景 | 建议措辞 |
|:-----|:---------|
| 论文含3+数据集对比 | 使用本表作为模板，标注泄漏模式 |
| 论文仅1个数据集 | 在Discussion中注明"泄漏模式可能因数据集而异" |
| 论文检测到Recall悖论 | 此发现本身有额外发表价值——"泄漏不仅能虚高指标，还能逆转临床敏感度方向" |

## 实验代码

```python
# 三数据集的统一消融协议
def three_dataset_comparison():
    datasets = {
        'PIDD': load_pidd(),   # 768×8, 34.9%
        'Heart': load_heart(), # 303×13, 45.9%
        'WDBC': load_wdbc(),   # 569×30, 37.3%
    }
    for name, (X, y) in datasets.items():
        helfix = run_cv(X, y, mode='helix')
        leaky = run_cv(X, y, mode='severe_leakage')
        print(f'{name}: ΔF1={leaky.f1-helix.f1:+.4f}, ΔRec={leaky.rec-helix.rec:+.4f}')
```

## 陷阱

- **只有2个数据集时"原则"证据不够**：PIDD vs WDBC 只能展示两极，缺少中间带。加入 Heart 后"难度比例损伤原则"的叙事从"比例"变为"非线性"——难度高时均匀膨胀，极高时无影响
- **Recall悖论只出现在特定模型**：PIDD 的 Recall Paradox (F1↑ Rec↓) 只在 Ensemble (GBC+LDA+SVC) 中出现，LogisticRegression 上不出现。写 Discussion 时需注明这种模型依赖性
- **绝对膨胀量 vs 相对膨胀量**：Heart 的绝对 ΔF1 (+0.1117) 最大，但相对基线更低。报告时应同时报两者
