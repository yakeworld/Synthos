# docx 引用格式提取 — 2026-06-07 实战

## 问题

用户发送 .docx 格式论文，需要提取正文中所有 `[N]` 格式引用来做 D8/D10a 检查。

## 错误方法

```python
# 只匹配单个数字
refs = re.findall(r'\[(\d+)\]', text)
# 结果：只匹配 [1], [2], [3]... 但漏掉 [5, 6] 中的 5 和 6
# 原因：[5, 6] 中 `[(\d+)` 匹配到 5 后，下一个匹配从 `]` 之后开始
```

## 正确方法

```python
# 先匹配整个 [...] 内容，再从中提取所有数字
refs_raw = re.findall(r'\[([^\]]+)\]', text)
refs = set()
for r in refs_raw:
    for num in re.findall(r'\d+', r):
        n = int(num)
        if 1 <= n <= 40:  # 根据实际引用范围过滤
            refs.add(n)
```

## 验证

提取后检查：
1. 所有 引用号 都在 1-N 范围内（N = bib 条目总数）
2. 无僵尸（bib 中有但正文没引用的）
3. 无孤儿（正文引用了但 bib 中没有的）

## 注意事项

- .docx 中引用可能是超链接格式，`paragraph.text` 提取后可能丢失格式但保留 `[N]` 文本
- 部分 docx 使用上标格式（`^N^`）而非 `[N]`，需额外匹配：`re.findall(r'\^(\d+)\^', text)`
- 如果正文中引用超过 40，检查是否有引用编号错误（如 [41] 但 bib 只有 40 条）
