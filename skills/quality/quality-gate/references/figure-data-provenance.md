# Figure Data Provenance Checklist

**凡图必源** — L0.5数据诚实门扩展：检查每张图是否有可追溯的生成路径。

## 三问验证
1. 这张图由哪个脚本生成？
2. 该脚本的数据源路径是否还存在？
3. 脚本输出的数值与论文中对应声明是否一致？

## 检查清单

### 图→脚本映射
- [ ] 每个figure文件有对应的生成脚本（grep figure_name *.py）
- [ ] 脚本中所有硬编码路径指向真实存在的目录
- [ ] 脚本可完整运行（python3 script.py 无错误）

### 数据→脚本一致性
- [ ] 脚本读取的数据源是论文声明的版本
- [ ] 数据预处理步骤已记录在代码中
- [ ] 所有CSV由代码生成（非手工编辑）

### 脚本→图一致性
- [ ] 图中数值与同一数据源的统计表一致
- [ ] 误差棒/显著性标注与统计检验p值方向一致
- [ ] Y轴范围不夸大/缩小组间差异
- [ ] 图例颜色命名不误导（"蓝色=我们的模型"但实际不是）

## 常见陷阱

### 1. 脚本路径腐烂
症状：目录重构后脚本静默失效，旧输出文件仍在造成"脚本工作正常"假象。
修复：每次交付前 grep -n "os.path.join|/figures/" *.py 检查路径存活。

### 2. 重复函数覆盖
症状：同一文件两个同名函数，Python使用最后定义的。
检测：grep -n "^def " script.py | sort — 同名出现两次需删除。

### 3. 数据源移动
症状：脚本报 FileNotFoundError 但旧图仍在。
修复：定期完整重跑所有脚本验证可复现性。
