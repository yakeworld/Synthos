# 证明材料（配套材料）编写指南 — 中国竞赛

## 要求概要

竞赛通知常见要求（如全球数智教育创新大赛 AI for Medicine 赛道）：
1. **可公开访问的测试账号** — GitHub 开源仓库 + 部署指南
2. **原创主体声明** — 研发团队、院校名称
3. **知识产权证明**（软件著作权、专利证书、论文）**或**研发过程佐证（技术路线图、代码记录）
4. **盖章** — 原始版本须经教务处/教师教学发展中心审核盖章

## 双版本管理

| 版本 | 上传入口 | 个人信息 | 盖章 |
|------|----------|----------|------|
| 原始版本 | 大赛管理系统（材料审核） | 含姓名/机构 | ✅ 需要 |
| 匿名版本 | 评审平台（专家评审） | 隐去 | ❌ 不需要 |

### 匿名化清单
- 去掉负责人姓名、职称
- 去掉具体院校/实验室名称 → 改为"医学人工智能研究团队"
- 去掉成员姓名表 → 改为"角色+贡献"描述
- 去掉公章栏
- 保留：GitHub 仓库、MIT 许可、论文标题（不含作者）
- 标题注明"匿名版本"

## 标准文档结构
1. **测试账号** — GitHub 仓库 + 部署命令 + 测试数据集
2. **原创主体** — 研发单位、团队贡献、原创性声明
3. **知识产权** — 开源记录（GitHub/许可证/代码记录）、论文/专利
4. **研发佐证** — 技术路线图（6层架构ASCII图）、原子架构图（I/O契约）、核心指标（进化轮/质量分/吸收数）、质量保障（闸门/双检/数据诚实门）
5. **时间线** — 里程碑表格
6. **测试流程** — 快速体验步骤
7. **审核栏**（仅原始版本）
