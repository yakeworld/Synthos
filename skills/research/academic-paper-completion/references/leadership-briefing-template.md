# BCI/AI Leadership Briefing Template

## Structure (5-point user-preferred outline)

```
# 脑机接口与人工智能方向：核心技术成果与应用展望

## 一、[技术名称] 技术定义与战略定位
- 技术定义（定位为BCI/脑科学方向）
- 技术背景（为什么现在做、填补什么空白）
- 战略意义（对比现有方案，强调落地可行性）
- 表格：与侵入式BCI/EEG-BCI的多维度对比

## 二、自研终端/系统：[系统名称]
- 硬件/软件实现描述
- 关键能力表格（模块 | 能力 | 状态）
- 技术架构图

## 三、基于 [平台名称] 的智能模型/系统
- 平台能力
- 技术管线
- 核心价值（量化指标）

## 四、[平台名称] 自主进化平台
- 平台定位
- 核心架构
- 技术特点
- 项目真实数据（仅使用可追溯的指标）

## 五、成果落地与应用范围
- 应用方向表格
- 合作模式
- 需要领导支持的方向
```

## Data Integrity Checklist

Before presenting any number in a leadership report:

| Question | Action |
|:---------|:-------|
| Does this number come from project data? | Use it, cite the file |
| Does this number come from a paper I've read? | Use it, cite the DOI |
| Is this number an estimate or guess? | Replace with qualitative language |
| Would the user be offended if this number is wrong? | Remove it immediately |

## Prohibited Number Types

These types of numbers are commonly fabricated and must be avoided:
- Market size estimates (e.g., "2000万患者", "5000万门诊量")
- Hardware specs without documentation (e.g., "0.1°精度", "60fps")
- Algorithm performance claims without benchmarks (e.g., "IoU > 0.92")
- Latency/throughput without profiling (e.g., "< 10ms")
- Any "improvement factor" not sourced from project data
