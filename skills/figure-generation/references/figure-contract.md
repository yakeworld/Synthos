# Figure Contract

画任何图之前，先建立契约。目标是让图表服务于论文的科学逻辑，而非让审美驱动。

## 必建契约

每次画图前在推理空间中建立契约：

```
结论Claim: [一句可证伪的主张，带动词]
图表原型: [quantitative-grid | schematic-led-composite | image-plate+quant | asymmetric-mixed-modality]
目标期刊/出口: [Nature | Science | Cell | NeurIPS | SCI | PPT | 竞赛]
后端: Python
最终尺寸: [单栏89mm | 双栏183mm | 自定义]
面板映射:
  a: [图a解决的科学问题]
  b: [图b解决的科学问题]
  c: [图c解决的科学问题]
证据层级:
  主证据: [英雄面板]
  验证证据: [支撑面板]
  对照/稳健性: [辅助面板]
所需统计: [样本量、误差棒类型、检验方法]
源数据: [CSV路径/列名/行数]
图像完整性备注: [全局调色? 裁剪?]
审稿风险: [审稿人最可能质疑的三件事]
```

不从模板开始。从结论开始，选择使结论清晰可辨的最少面板集。

## 结论Claim规则

- 结论是一个带动词的句子："Treatment X reduces Y by restoring Z"，而不是"Treatment results"
- 每个面板回答一个独特的问题。遮住任一面板如不削弱论证，删除或合并
- 主要证据与支持证据分开。主要证据得到英雄面板或最清晰轴；对照面板视觉上安静
- 如果用户提供数据但没有主张，从数据推断临时主张并确认

## 原型快速决策

| 条件 | 选 |
|------|----|
| 主要是数值比较 | quantitative-grid |
| 需要先理解流程/机制/装置 | schematic-led-composite |
| 成像/显微/组织学/电泳带头 | image-plate+quant |
| 混合示意图+光栅+热图+定量 | asymmetric-mixed-modality |

## 面板冗余检查

- [ ] 面板b不是面板a的数据以不同形式展示
- [ ] 面板c增加了a和b中没有的维度
- [ ] 每个面板有独立的轴标签词汇
- [ ] 遮住任一面板不削弱论证 → 删除

## 审稿风险提示

提交前检查：

- [ ] 样本量是否在图例或源数据中可见？
- [ ] 误差棒、区间和统计检验是否定义？
- [ ] 可比较面板间的范围是否可比？
- [ ] 代表性图像是否量化并可追溯到源文件？
- [ ] 图像调整是否全局并记录？
- [ ] 能否用更少面板得出相同结论？
