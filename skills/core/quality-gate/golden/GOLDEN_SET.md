# Quality Gate — Golden Set

> 质量闸门通过的标准测试用例集。每个检查项有明确的"通过"标准。

## Golden Test 1: L0 动灵方向检查

**输入**: 系统当前生长方向 + 交付物方向
**预期输出**: aligned=true 或 aligned=false + 原因
**通过标准**: 
- 输出包含明确的 aligned 字段
- 如果 aligned=false，必须说明方向不一致的具体原因
- 方向不对的交付物不进入技术检查

## Golden Test 2: L0.5 数据诚实门

**输入**: 论文中所有可验证数据声明
**预期输出**: 每个声明的源文件检查结果
**通过标准**:
- 每个有数字/实验结果的声明都有源文件路径
- 无源文件的声明被标记为"需删除"或"理论推算"
- 输出包含 unsupported_claims 列表

## Golden Test 3: G5 引用质量 — 形式检查

**输入**: 论文bib条目
**预期输出**: D10a, 孤儿数, 僵尸数, DOI覆盖率
**通过标准**:
- D10a ≥ 95%
- 孤儿引用 = 0
- 僵尸引用 = 0
- DOI ≥ 90%
- 全部通过才记为 passed=true

## Golden Test 4: G5 引用质量 — 实质检查

**输入**: 论文引用上下文 + 参考文献PDF
**预期输出**: 逐篇审查报告，判断引用是否得当
**通过标准**:
- 引用文献真的支持论文中的论断
- 无张冠李戴（引用了A期刊但实际内容是B主题）
- 无断章取义（只引用了文献的一部分而忽略整体结论）
- 引用恰当率 = 100%

## Golden Test 5: G1-G7 闸门递进

**输入**: 论文在各阶段的状态
**预期输出**: 递进的 pass/fail 链
**通过标准**:
- G1不通过不得进入G2
- G2不通过不得进入G3
- G3不通过不得进入G4
- G4不通过不得进入G5
- G5不通过不得进入G6
- G6不通过不得进入G7
- G7通过才进入提交材料准备

## Golden Test 6: 一维一渡

**输入**: 质量检查请求
**预期输出**: 单次检查只聚焦一个等级
**通过标准**:
- L1检查只检查响应级质量
- L2检查只检查项目级质量
- L3检查只检查管线级质量
- 不跳步不并行

## Golden Test 7: 无录不过

**输入**: 质量门执行记录
**预期输出**: 有skill_view调用记录
**通过标准**:
- 每次质量门检查都有skill_view调用
- 无skill_view记录的检查视为未执行
- 门不通过
