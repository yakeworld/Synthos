# Golden Test Set — quality-gate

> 质量闸门技能的黄金测试方法论。不测试单项检查的正确性，而是验证**整个报告体系**的完整性和一致性。

## 测试对象

quality-gate 在给定论文目录上生成四份并行报告时，必须满足以下条件：

## 测试一：四报告完整性

**输入**：一篇论文目录（含 paper.tex + 03-code/*.json + 06-references/ + state.json）

**期望输出**：
- [ ] 报告一：通用六域 → 含 Q1-Q6 六个域的 PASS/SOFT/HARD 判定
- [ ] 报告二：类型专项 → 按 L0 分类器加载正确模板
- [ ] 报告三：参考文献审查 → 引用功能分类 + 引文网络
- [ ] 报告四：检查员报告 → 凡数必源矩阵 + 凡引必查清单 + 代码诚实验证 + 虚构检测扫描

## 测试二：数值可追溯性（凡数必源）

**输入**：论文 paper.tex 中的所有数值声明

**期望输出**：
- [ ] 可追溯率 ≥ 95% → 判定 PASS
- [ ] 可追溯率 80-95% → 判定 SOFT
- [ ] 可追溯率 < 80% → 判定 FAIL
- [ ] 每个不可追溯数值标注具体位置和原因
- [ ] paper_claims/actual_values 双结构 → 标记 FABRICATION_DETECTED

## 测试三：引用验证（凡引必查）

**输入**：references.bib 中的所有条目 + 06-references/ 下的 PDF 文件

**期望输出**：
- [ ] 通过率 ≥ 90% → PASS
- [ ] 通过率 < 90% → SOFT/FAIL
- [ ] 每个 PDF 都进行了三检：存在性 + DOI匹配 + 体裁验证

## 测试四：虚构检测覆盖率

**输入**：已完成修复的论文目录

**期望输出**：至少检测以下 7 种虚构模式
- [ ] paper_claims/actual_values 双结构
- [ ] Ensemble 成员在 JSON 间不一致
- [ ] F1/Acc 绝对值异常（>95%）
- [ ] 所有指标同时最优
- [ ] 引用键在 bib 但无 PDF
- [ ] 统计量在代码中无记录
- [ ] 叙事-数据比例失调

## 测试五：报告互不冲突

**输入**：同一篇论文目录

**期望输出**：
- [ ] 四份报告判定独立，互不影响
- [ ] 最终判定取四份报告中最严格的
- [ ] 附录合并所有报告的 P0/P1/P2 问题
- [ ] 问题清单按严重度排序，不重复
