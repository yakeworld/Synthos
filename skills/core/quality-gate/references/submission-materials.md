# 投稿材料准备清单和模板

## 通用要求

Elsevier期刊（如Measurement、Pattern Recognition等）投稿需要以下材料：

### 必需材料

| 材料 | 要求 | 说明 |
|------|------|------|
| Manuscript PDF | 主稿PDF文件 | 35页以内为佳，黑白/彩色均可 |
| Graphical Abstract | 单页图文件 | 150mm宽，展示核心贡献 |
| Highlights | 5条要点 | 每条85字符以内 |
| Cover Letter | 投稿信 | 说明贡献、匹配度、无发表承诺 |
| Author Information | 作者信息 | ORCID、联系方式、单位 |
| CRediT Statement | 作者贡献 | 已在论文.tex中包含 |
| Conflict of Interest | 利益冲突声明 | 已在论文.tex中包含 |
| Data Availability | 数据可用性声明 | 已在论文.tex中包含 |

### 可选材料

| 材料 | 说明 |
|------|------|
| 推荐审稿人 | 3位合适领域的审稿人 |
| 回避审稿人 | 有利益冲突的审稿人 |
| 亮点图说明 | Graphical Abstract的文字描述 |
| 代码可用性 | 代码开源计划 |

## Highlights要求

Elsevier要求：
- 最多5条要点
- 每条**不超过85个字符**（含空格）
- 要点应突出创新性、实验结果、应用价值

写作模板：
1. 方法创新（1条）— "Introduces [novel method] leveraging [key technique] to solve [problem]"
2. 实验结果（1-2条）— "Achieves [metric=value] on [dataset], significantly outperforming [baseline]"
3. 应用价值（1条）— "Opens new possibilities for [application domain] that relies on [capability]"
4. 技术优势（1条）— "Employs [technique] for [purpose], demonstrating [advantage]"

## Graphical Abstract设计规范

- **宽度**: 150mm（单栏）或 325mm（半栏）
- **格式**: PDF、EPS、TIFF、JPEG（高分辨率）
- **内容结构**: 问题 → 方法 → 结果 → 应用
- **风格**: 简洁、专业、学术风格
- **字体**: Arial/Helvetica，清晰可读
- **颜色**: 3-5种颜色以内，黑白打印也能区分

内容布局建议：
- 左/上：问题陈述（当前方法的局限）
- 中：我们的方法（核心创新）
- 右/下：主要结果（关键指标）+ 应用场景

## 推荐审稿人选择原则

1. **领域匹配**: 审稿人研究领域与论文高度相关
2. **地理多样性**: 不集中在同一国家/地区
3. **无利益冲突**: 近3年未与作者合作过
4. **知名度**: 领域内有影响力的学者
5. **活跃度**: 近年有相关论文发表

## 提交前最后检查

在提交系统入口，逐项检查：
- [ ] Manuscript PDF可打开、内容完整
- [ ] Graphical Abstract尺寸符合要求
- [ ] Highlights不超过5条、每条≤85字符
- [ ] Cover Letter所有占位符已替换
- [ ] 作者信息完整（ORCID、邮箱、单位）
- [ ] CRediT、COI、Data Availability已在.tex中包含
- [ ] 参考文献格式正确（Bib已编译为BBL）
- [ ] 所有图片在PDF中清晰可读
- [ ] 论文标题与投稿系统输入一致
- [ ] 通讯作者邮箱正确
