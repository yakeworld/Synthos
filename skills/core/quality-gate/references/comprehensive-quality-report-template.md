## 报告生成流程

1. **收集数据**: 同时检查 paper.tex, all JSON 实验输出文件, paper.log, paper.bbl, state.json, 05-figures/, 03-code/
2. **state.json 交叉验证**: 对比 top-level quality_score vs gates_result.quality_score vs 独立审计
3. **数值可追溯矩阵**: 逐行遍历 paper.tex 中的每个数字声明，与所有 JSON 源交叉对比
4. **多JSON源 ensemble 检测**: 从所有 JSON 中提取 ensemble 成员，与论文声称比对
5. **P0/P1/P2 分类**: 数据诚实问题→P0, 格式/引用→P1, 可选项→P2
6. **生成报告**: 按上述结构输出

## 实战参考

- PIMA-CRISP-DM (2026-06-24): state.json 自报96 vs gates_result 85 vs 独立审计79
- PIMA-CRISP-DM (2026-06-24): Ensemble 三源不一致 (paper: CatBoost+GBC+LR, ablation JSON: GBC+LDA+SVC, comprehensive JSON: LDA+GBC+LR)

## 报告生成后的必做动作

> **质量报告不是终点，是修复的起点。** 报告生成后，应立即执行自主修复闭环：
> 1. 逐条分析 P0/P1 问题，分类为「可自主修复」和「需人工」
> 2. 并行执行所有可自主修复的操作（patch paper.tex、清理 bib、重编译）
> 3. 写 fix-log.md，标记待办
> 4. 更新 state.json
> 5. 发送已修复的 PDF 给用户（而不是扔个问题清单让他们自己去修）
>
> 详见 `codex-comprehensive-quality-report-workflow.md` 的 Step 5 自主修复闭环。
