### 致命错误模式 (L0.5数据诚实门)

1. **学历错误**: PPT写"医学博士"，实际硕士 → 这是硬事实错误，不可近似
2. **专利号虚构**: `ZL 2024 1 0123456.7` 是序列号，不是真实专利号格式 → 答辩时专家查证即出局
3. **版本矛盾**: 同一PPT出现两套Synthos版本数据 (v2.19.0/67轮 vs v4.3.0/53轮)
4. **数据口径不一致**: 专利数(8vs7)、经费(300万vs455万)、团队规模(18人vs33人)

### 修复优先级

1. **L0.5致命错误先修** → 学历、专利号、版本矛盾
2. **数据一致性** → 对照原始申报书逐条核对所有数值
3. **结构优化** → 删除冗余页面，控制每页≤200字
4. **完整性补充** → 补充联系电话、IF标注、核心创新点

### 审查检查清单

| 检查项 | 验证方法 | 标准 |
|:-------|:---------|:-----|
| 学历 | 对照申报书/人事档案 | 必须精确，不可近似 |
| 职称 | 对照申报书 | 必须精确 |
| 专利号 | 对照专利数据库/申报书 | 格式：ZL+18位数字，不可虚构 |
| 经费 | 对照申报书项目明细 | 口径一致 |

| 论文数 | 对照申报书 | 数量一致 |
| 版本信息 | 对照系统状态文件 | 版本+轮数+评分一致 |
| 核心创新 | 对照申报书"标志性成果" | 关键创新点必须出现 |
