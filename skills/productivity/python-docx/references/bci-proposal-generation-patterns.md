## 术语修正（⚡P0 — 已纠正多次）

| 错误表述 | 正确表述 | 原因 |
|---------|---------|------|
| 完稿手稿 / 手稿管线 | 已完稿技术成果 / 技术成果管线 | 未发表≠论文，统一称为技术成果 |
| 博士生导师 / 博导 | 硕士生导师 / 硕导 | 杨晓凯仅硕士生导师 |
| 优势项中的"临床数据" | 删除（人人有数据） | 无差异化 |
| 优势项中的"人才储备" | 改为"人才团队" | 避免暗示 |
| 优势项中的"政策定位" | 删除（人人有政策） | 无差异化 |

## 设备表格规范

### 统一列结构（三列）
`["项目", "参数", "说明"]` — 无空列，每列必须填内容。禁止使用空字符串。

### 刷新率
- 眼动仪：**≥90Hz**（非"90Hz"）
- 眼动视频：≥90fps（非"@90fps"）

### 预算表
5列：`["项目", "内容", "数量", "金额(万元)", "占比"]`
每项单独一行，不堆叠。占位符填"—"。

## 生成流程
1. 加载 eye-tracking-platform 获取K230完整规格
2. 加载 bci-wenzhou 获取用户资产和政策对齐
3. 加载 competition-submission 获取政策对接策略
4. 加载 powerpoint 获取docx生成模式
5. 严格遵循术语修正和表格规范
6. 生成后逐项检查
