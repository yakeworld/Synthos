## 核心流程

```
通知PDF → OCR提取要求 → 多项目四维评审 → 选定项目 → 
填写报名表 → 起草商业计划书 → 收集团队成员信息 → 
补充字段 → 发件
```

## Step 1: 通知OCR

扫描版PDF通知 → `pdftoppm` → `tesseract -l chi_sim`

## Step 2: 多项目四维评审

对候选项目从四个维度评分（每项1-10）：

| 维度 | 评分标准 |
|:-----|:---------|
| 创新性 | 技术独特性、学术支撑强度 |
| 实用性 | 临床刚需程度、应用场景明确性 |
| 市场潜力 | 市场规模、竞争格局、替代空间 |
| 团队实力 | 论文积累、临床资源、技术壁垒 |

输出排序表，推荐最高分项目+组合策略建议（核心硬件+整合算法模块）

## Step 3: 团队成员信息收集

### 信息来源优先级

| 来源 | 文件格式 | 工具 |
|:-----|:---------|:-----|
| 论文Authors.docx | docx | `libreoffice --headless --convert-to txt` |
| 人员邮箱表 | xlsx | `openpyxl` |
| 医院全院信息数据库 | xls（旧格式） | `libreoffice --headless --convert-to xlsx` → `openpyxl` |
| 论文作者列表 | tex/md | `grep` / `python3 re` |
| 桌面article_todo目录 | 目录结构 | `find` + 文件名匹配 |

### 关键命令

```bash
# docx → txt
libreoffice --headless --convert-to txt Authors.docx --outdir /tmp/

# xls → xlsx（旧格式转换）
libreoffice --headless --convert-to xlsx 全院信息.xls --outdir /tmp/

# 读取xlsx
python3 -c "
import openpyxl
wb = openpyxl.load_workbook('/tmp/全院信息.xlsx')
ws = wb.active
for r in range(2, ws.max_row+1):
    name = ws.cell(r, 5).value
    if name and '杨晓凯' in str(name):
        for c in range(1, 33):
            print(f'{ws.cell(1,c).value}: {ws.cell(r,c).value}')
"

# 搜索article_todo目录
grep -r "姓名\|作者\|author" ~/桌面/article_todo/*/paper.md 2>/dev/null
```

## Step 4: 报名表结构

报名表包含：
1. **机构/团队信息** — 名称、信用代码、联系人
2. **核心人员信息** — 每人姓名/年龄/职务/电话/邮箱/学历/毕业院校
3. **项目信息** — 名称、类别（医工交叉概念期/融资发展阶段/解决方案）
4. **项目概况** — 文本描述（痛点+方案+创新亮点）
5. **发展需求** — 多选（专家辅导/融资/推广/落地）+ 需求描述

## Step 5: 商业计划书结构

大赛要求9大章节：

| 章节 | 内容要点 |
|:-----|:---------|
| ①项目介绍 | 产品形态、技术架构、适用场景、路线图 |
| ②行业市场 | 市场规模、目标细分、政策利好 |
| ③核心技术 | 算法模块、技术壁垒、知识产权 |
| ④竞争分析 | 国内外竞品对比表、差异化优势 |
| ⑤商业模式 | 收入模型、定价、渠道、合作伙伴 |
| ⑥团队介绍 | 核心成员姓名/角色/背景 |
| ⑦财务预测 | 启动资金、3年营收、成本结构 |
| ⑧风险控制 | 技术/市场/法规风险+应对+退出机制 |
| ⑨融资方案 | 本轮金额/股权/用途、后续轮次 |

## 陷阱

1. **全院信息数据库可能含同名不同人** — 邵琪琪（护士，1996，大专）≠ 邵琪琪（硕士，1998，江西理工）。必须用身份证号交叉验证
2. **Authors.docx中作者名与团队名可能有差异** — 张一涵 vs 张祎函，以用户确认为准
3. **论文中出现的合作者不等同于团队成员** — 周逸飞是论文co-author但不是参赛团队成员
4. **商业计划书数据合理性** — 财务预测的3年营收增长曲线需有逻辑支撑
5. **"已发表"状态误用** — Synthos内部质量通过的论文并非"已发表"，须逐篇确认发表状态后再写入商业计划书。用户对"已发表"vs"已完成"vs"已投稿"的区分高度敏感
6. **专利数低估** — 从网站提取专利数据时，必须用HTML `<tr>`表格行解析（非仅ZL/CN号正则），否则会漏掉表格后半部分的专利条目。2026-05-30实战：仅正则匹配得9项，用表格行解析得27项
7. **市场数据无来源则改定性** — 无明确来源的具体市场规模数字（如"15亿元"、"增长率18%"）应改为定性表述（如"市场空间较大"），或标注"据行业调研"
