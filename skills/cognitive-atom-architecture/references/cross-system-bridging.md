# 跨系统桥接模式

> 会话: 2026-06-07
> 模式: 两个独立系统之间的双向连接
> 实例: Synthos论文管线 ↔ AKNE知识图谱

## 核心原则

1. **注入非复制** — 不是文件拷贝，而是语义映射（目录→节点，分类→边）
2. **类型区分** — 注入的节点使用独立 type 前缀（`synthos_paper` 而非 `entity`）
3. **增量安全** — 同名节点不重复，边不删除，幂等运行
4. **领域先行** — 基于领域关键词建立连接，而非盲目全连
5. **查询可反** — 注入的同时创建反向查询能力

## 典型数据流

```
Synthos (系统A)                  AKNE (系统B)
┌─────────────────┐             ┌────────────────────┐
│ outputs/papers/ │ ─注入───→  │ graph.json         │
│ skills/         │ ─索引───→  │ sources/synthos/   │
│ evolution/      │ ─引用───→  │ wiki/projects/     │
└─────────────────┘             └────────────────────┘

反向查询:
  AKNE实体 "BPPV" ─查询──→  Synthos论文 15篇
  AKNE概念 "前庭解剖" ─查询──→  Synthos技能 knowledge-acquisition
```

## 最小可行桥接

最少需要的文件：
1. 一个 Python 脚本：注入 + 查询
2. 一个目录结构：AKNE 侧新增 sources/<系统A>/index.md
3. 一个 Wiki 页面：AKNE 侧新增 wiki/projects/<系统A>.md

## 领域关键词设计

关键词应满足：
- 足够具体：`"bppv"` 而非 `"医学"`
- 覆盖中英：`"bppv"` + `"前庭"` + `"耳石"`
- 不冲突：`"segment"` 在医学影像域 vs 通用领域要区分
- 可验证：每个关键词都能在系统A的实际文件名中找到

## 边权重策略

| 权重 | 条件 |
|------|------|
| 1.0 | 直接匹配（AKNE内部 references 边） |
| 0.3-0.8 | 领域关键词重叠 |
| <0.3 | 跳过（避免噪声） |

## 边去重

使用 `(source, target, type)` 三元组作为键，不添加重复边。
