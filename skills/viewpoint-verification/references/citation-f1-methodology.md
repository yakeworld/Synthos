# Citation F1 — P0/P1 引用质量评价方法论

**来源**: PaperOrchestra (Song et al., 2026, arXiv:2604.05018, App. F.3)
**吸收类型**: P1 — 证据质量增强（注入 viewpoint-verification 原子）
**原理**: 将生成论文的引用列表与 ground truth 对比，计算 Precision/Recall/F1

---

## 动机

Synthos 的 viewpoint-verification 原子原有的置信度计算基于**反方观点强度** × **内部弱点** × **鲁棒性关切**。但这个计算有一个盲区：**引用质量**。

一个假设即使有大量反方观点支持，如果它的引用列表遗漏了关键文献（P0），它的可信度也应降低。Citation F1 填补了这个缺口。

## P0/P1 分类定义

| 优先级 | 定义 | 示例 |
|--------|------|------|
| **P0 (Must-Cite)** | 论文中**必须引用**的核心文献 | 实验对比的baseline、使用的数据集、直接继承/修改的核心方法、依赖的标准指标 |
| **P1 (Good-To-Have)** | 补充性引用 | 标准背景综述、一般性相关工作、次要工具/实现 |

## 引用质量评分程序

### Step 1: 分类引用

使用以下 prompt 模板，对论点支持的引用列表进行 P0/P1 分类：

> 你是一名严格的学术评审专家。阅读以下论文论证和其引用的参考文献列表。
> 将每个引用分类为两个优先级：
>
> **P0 (Must-Cite)** — 必须引用。包括：
> - 实验直接对比的 baseline
> - 论文使用或评估的数据集
> - 论文直接继承或修改的核心方法
> - 论文高度依赖的标准指标或基准数字
>
> **P1 (Good-To-Have)** — 建议引用。包括：
> - 覆盖广泛历史的标准背景文献
> - 不直接竞争或不直接构建的一般相关工作
> - 次要工具或浅层提及的实现
>
> 返回一个 JSON 对象：{"1": "P0", "2": "P1", ...}

### Step 2: 计算引用质量分数

对于每个假设/论点，其引用列表被分为：

```
P0_set = {P0引用的标识符}
P1_set = {P1引用的标识符}
```

引用质量得分 = (P0引用数 + 0.5 × P1引用数) / (P0引用数 + P1引用数)

- 如果引用列表为空：引用质量 = 0.0
- 如果全部是 P0：引用质量 = 1.0
- 如果 P0 > P1：引用质量 > 0.75

### Step 3: 置信度修正

引用质量得分作为置信度修正因子，与其他因子结合：

```
修正后置信度 = 原置信度 × min(1.0, 引用质量 + 0.3)
```

| 场景 | 原置信度 | 引用质量 | 修正后 | 影响 |
|------|:--------:|:--------:|:------:|:----:|
| 优秀引用 | 0.85 | 0.90 | 0.85×1.0=0.85 | 无变化 ✅ |
| 中等引用 | 0.70 | 0.60 | 0.70×0.90=0.63 | 降 -0.07 |
| 差引用 | 0.65 | 0.30 | 0.65×0.60=0.39 | 降 -0.26 🔴 |

## 与现有置信度计算整合

在 viewpoint-verification 的置信度计算步骤（当前 Step 3e）之后增加：

```
Step 3e.5: 引用质量修正（PW-Bench吸收）
  - 提取当前假设/论点引用的所有参考文献
  - 对引用列表执行 P0/P1 分类
  - 计算引用质量得分
  - 应用置信度修正：confidence = confidence × min(1.0, citation_quality + 0.3)
  - 在验证记录中添加 "citation_quality" 字段
```

## 与黄金标准金集（Golden Set）的关系

P0/P1 分类本质上是**判断性的**（一个引用是 P0 还是 P1 取决于论文的论述焦点）。因此引用质量评分应在以下约束下使用：

1. **不参与自动裁决** — 引用质量修正不应单独触发 verdict 变化
2. **仅作为置信度弱信号** — 权重低于反方观点强度和内部弱点
3. **在验证记录中透明报告** — 让人类用户判断引用质量评价是否合理
