# 政府技术申报方案质量审查 — 方法参考

> 基于2026-06-23温州市人民医院BCI试点医院申报方案审查实战

## 审查流程

### 1. 文档扫描与分类
- 确定文档类型（BCI试点申请 / 科研平台申报 / 人才计划 / 标准制定提案等）
- 提取关键数据点清单：所有数字/百分比/性能指标/成本数据
- 标注重叠声明的段落（摘要/正文/总结中同一数据的多轮出现）

### 2. 数据准确性验算

#### 成功案例：BCI方案成本数据
- 原文：单套硬件成本2500元
- 验证：K230芯片BOM ~$5-10 + 3×MIPI摄像头 ~$30-60 + PCB/结构件/组装 ~$50-100 = 约$100-170 ≈ 720-1220元 (物料)
- 结论：2500元作为物料成本偏高但含margin合理 → 修正为"物料成本约2500元"
- 进口设备对比：Tobii Pro Spectrum ~20万, EyeLink 1000 ~15-30万 → 5-30万区间合理

#### 失败案例：BPPV论文数量
- 初始错误：仅查活跃管线（5篇）→ 误判"14篇"为虚数
- 修正后：PubMed(yakeworld@wmu.edu.cn)查到14篇已发表 + 存档10篇 + 管线5篇 = 29篇储备
- 教训：**成果数量验证必须查三个来源** — 管线(含存档) / PubMed / 投稿文件汇总

### 3. 技术论证逻辑检查

| 检查维度 | BCI案例中的发现 | 修正 |
|:---------|:----------------|:-----|
| SNR对比 | EOG 100:1 vs 侵入式BCI 1:10 — 物理尺度不同，直接比误导 | 改为与头皮EEG对比（同为非侵入式） |
| 带宽计算 | 三路1080p90 ~1.07GB/s，K230 DDR带宽由CPU/NPU/ISP共享 | 改为"单路最高" + 场景化配置说明 |
| 覆盖范围 | "一个平台同时服务六大维度" | 分已验证(眩晕)/验证中(ADHD/抑郁) |
| 成果定义 | "64项完稿"定义模糊 | 明确为"94项技术储备（含64项已整备成果）" |

### 4. SNR对比修正话术（关键技巧）

**原文问题**：EOG SNR(100:1) vs 侵入式BCI SNR(1:10) → 看似气势足但属于"苹果比橘子"

**正确论述逻辑**：
```
眼动追踪作为非侵入式BCI方案
  → 信号源：眼外肌宏观电场(mV级)
  → 噪音基底极低（角膜-视网膜电势差稳定）
  → SNR可达100:1，确保单次采集即稳定
  → 对比基准应是常规头皮EEG（同为非侵入式）
  → SNR远高于EEG → 无需电磁屏蔽室 → 门诊即用
  → 这才是"大规模临床部署"的真正优势
```

**不要做的事情**：
- ❌ 不要和侵入式BCI比SNR数字（比1:10高1000倍也不能证明方案更好）
- ❌ 不要说"远超侵入式BCI"（暴露对两种技术信号特性的理解不够）

### 5. 成本声明的分层表述

| 表述层级 | 适用场景 | 风险 |
|:---------|:---------|:-----|
| "硬件物料成本约2500元" | ✅ 技术方案正文 | 低 — 有BOM支撑 |
| "单套成本仅2500元" | ❌ 无限定 | 高 — 被追问"包含认证/质保/分摊吗" |
| "量产成本可控制在2500元以内" | ✅ 愿景展望 | 中 — 需明确是指量产后BOM |

**规则**：方案正文中第一次出现成本声明时必须加限定词（"物料"或"BOM"），后续引用可简化。

### 6. 内部一致性检查清单

- [ ] 摘要中的数据与正文表格一致
- [ ] 各章节间的同一术语定义一致
- [ ] 总结段的数据与各节一致
- [ ] 交叉验证表中同一个数据集的不同F1值有解释
- [ ] 标题中的"已完成"与正文的"正在验证"无冲突

### 7. python-docx修改工作流

```python
# 核心步骤
from docx import Document
doc = Document('源文件.docx')
for para in doc.paragraphs:
    if '目标段落特征文本' in para.text:
        # 清空所有run
        for run in para.runs:
            run.text = ''
        # 新文本写入第一个run（保留样式）
        para.runs[0].text = new_text
doc.save('修改版.docx')
```

**注意**：
- 段落文本可能跨多个run（格式片段），直接用`.text`属性读全文后替换所有run
- 替换后需验证：用python-docx重新读取检查每个修改点
- 扫描残留风险：搜索原始关键词确保无遗漏

## 实战数据：BCI方案-关键参数验算

### K230三路1080p90带宽
```
单路1080p90 (YUV422): 1920×1080×90×2 = 373 MB/s
三路合计: 1.07 GB/s
K230 LPDDR4带宽: ~3.2-6.4 GB/s (共享CPU/NPU/ISP)
结论: ISP三路同时满负荷需确认，实际部署灵活配置
```
