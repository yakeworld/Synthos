# 提案创新性评估：NotebookLM方法

> 2026-05-26 人才Token提案实战验证

## 流程

```
Step 1: 上传提案到已有的民进NotebookLM项目
  notebooklm use <project_id>
  cat <提案文件> | notebooklm source add --title '<提案标题>' -

Step 2: 等待source ready
  notebooklm source list | grep '<标题>'   # 检查status是否为ready

Step 3: 执行6维创新性评估
  notebooklm ask '请从以下维度评价新上传的提案《[标题]》的创新性：
  
  1. 选题新颖性：这个选题在现有民进提案/参政议政材料中是否见过？属于首创、跟进、还是重复？
  2. 视角独特性：提案人的背景给提案带来了什么独特视角？
  3. 方案原创性：核心机制设计是否有先例？
  4. 政策可行性：在现有政策框架上延伸的落地难度如何？
  5. 地方适配性：选择试点区域的理由是否充分？
  6. 总体评价：与现有提案相比，创新性如何？
  
  请逐条回答，给出具体理由和引用来源。'
```

## 6维评估框架

| 维度 | 评分范围 | 核心问题 |
|:-----|:---------|:---------|
| ① 选题新颖性 | ⭐1-5 | 此前是否有类似提案/建议？ |
| ② 视角独特性 | ⭐1-5 | 提案人专业背景提供了什么独特切入点？ |
| ③ 方案原创性 | ⭐1-5 | 解决方案的核心机制有先例吗？ |
| ④ 政策可行性 | ⭐1-5 | 落地难度和财政成本如何？ |
| ⑤ 地方适配性 | ⭐1-5 | 选择特定区域的理由是否充分？ |
| ⑥ 总体定位 | 定性 | 属于"破除存量问题""改良传统工作"还是"建构增量未来"？ |

## 实战效果

人才Token提案通过此评估，NotebookLM给出结论：创新性处于"最高维度"——前两份提案分别是"解决存量问题"和"改良传统工作"，人才Token提案是"建构增量未来"。

**关键**：NotebookLM能同时比较项目内的多份提案/文献，这个横向对比能力是人工难以实现的。
