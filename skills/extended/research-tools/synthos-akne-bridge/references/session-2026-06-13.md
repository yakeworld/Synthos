# AKNE 查询引擎 — Session 2026-06-13 诊断与修复记录

## 问题诊断

### 问题 1: resolve_entity 不存在于 KnowledgeGraph

**症状**: `kg.resolve_entity("BPPV")` → `AttributeError: 'KnowledgeGraph' object has no attribute 'resolve_entity'`

**根因**: `resolve_entity()` 定义在 `akne/reason/query_engine.py` 的 `QueryEngine` 类中，不在 `KnowledgeGraph` 类中。

**修复**: 所有代码必须通过 QueryEngine 调用:
```python
from akne.reason.query_engine import QueryEngine
kg = KnowledgeGraph(graph_path)
qe = QueryEngine(graph_index=kg)
entity = qe.resolve_entity("BPPV")
```

### 问题 2: find_related() 返回 3 元组，不是 4 元组

**症状**: `for n, r, m, d in related:` → `ValueError: too many values to unpack (expected 4)`

**根因**: `kg.find_related(entity)` 返回 `[(neighbor, relation_chain, metadata), ...]`，每个元素是 3 元组。

**修复**: 只解包 3 值:
```python
for neighbor, rel_chain, meta in kg.find_related(entity, depth=2):
    ...
```

### 问题 3: in_edges()/out_edges() 数据格式

**症状**: `for src, tgt, d in kg.graph.in_edges(p, data=True, keys=True):` → 返回 4 值 `(src, tgt, key, data)`

**根因**: `data=True, keys=True` 返回 4 元组，`data=True` 仅返回 3 元组 `(src, tgt, data)`。混用会导致解包错误。

**修复**: 统一使用一种格式:
```python
# 统一 3 元组
for src, tgt, d in kg.graph.in_edges(p, data=True):
    if isinstance(d, dict):
        ...

# 统一 4 元组
for src, tgt, key, d in kg.graph.in_edges(p, data=True, keys=True):
    ...
```

### 问题 4: execute_code 与 AKNE 环境隔离

**症状**: execute_code 中 `import networkx` 失败（venv 无 networkx），或 `import sentence-transformers` 失败。

**根因**: execute_code 运行在 venv Python 3.11 中（networkx 3.6.1 已装，但 sentence-transformers 在系统 Python 3.12）。

**修复**: 所有 AKNE 查询必须通过 terminal 工具走系统 Python 3.12。
```bash
# 正确
terminal: akne-query.sh simple "BPPV"

# 错误
execute_code: from akne.graph import ...
```

| 眩晕 | simple | 398 related sources |
| Synthos | bridge | 148 papers, 25 skills, 0 orphans |

### 架构决策记录

1. **venv 不装 torch**: sentence-transformers 依赖 torch (500MB+)，venv 已有 970MB，装后超 1.5GB 不可接受
2. **venv 装 networkx**: 仅 2MB，已安装 (3.6.1)，execute_code 可运行轻量图操作
3. **AKNE 查询走 terminal**: 系统 Python 3.12 有所有依赖（jieba + sentence-transformers + networkx）
4. **jieba 安装**: 系统 Python 用户 site-packages（~10MB），用于中文分词和多 token 匹配
5. **向量搜索**: vectors.db 533 条记录覆盖 500/1145 源文件，剩余源文件未向量化
6. **QueryEngine 多词查询**: 英文多词查询（"eye tracking methodology"）仍无效，依赖图搜索或向量搜索
