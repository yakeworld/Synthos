# 论文质量深度审查 — 工作流程清单

## 审查前准备

- [ ] 确认论文目录存在且结构完整
- [ ] 确认参考文献列表（bibkeys）完整
- [ ] 确认下载引擎可用（至少1个来源可用）
- [ ] 确认当前网络连接正常

## 审查流程

### Step 1: 文献下载与内容提取 (约1-2小时)

- [ ] 使用 pdf_download_engine.py 下载所有参考文献 PDF
- [ ] 逐篇提取每篇文献的：
  - 标题、作者、年份、期刊
  - 摘要（Abstract）
  - 方法（Methodology）
  - 主要结果（Key Results）
  - 结论（Conclusions）
- [ ] 提取每篇文献在论文中的引用理由（context）
- [ ] 记录每篇文献的下载来源和状态

### Step 2: 文献引用深度验证 (约2-3小时)

对每篇参考文献执行：

- [ ] **验证引用理由**：该文献是否真的支持论文中的引用主张？
  - 提取论文中的具体引用上下文
  - 检查文献是否真的包含该信息
  - 检查引用是否准确（有无断章取义）
  - 检查引用是否合理（有无误导性引用）
  
- [ ] **生成引用质量评分**（每篇 0-10 分）：
  - 10分：文献明确支持引用主张，无歧义
  - 8-9分：文献部分支持引用主张，有条件性
  - 6-7分：文献勉强支持引用主张，需解释
  - 4-5分：文献不支持引用主张，但可解释
  - 2-3分：文献明确不支持引用主张
  - 0-1分：文献不存在或完全无关

- [ ] **生成引用验证报告**：
  - 每篇文献的验证结果
  - 问题文献标注
  - 改进建议

### Step 3: 研究空白验证 (约1-2小时)

- [ ] 识别论文的 Introduction 逻辑链：
  - 背景（Background）：论文声称的研究背景
  - 缺口（Gap）：论文声称的研究缺口
  - 空白（Gap）：论文声称的研究空白

- [ ] 验证每个空白声明：
  - 独立检索最新文献验证空白是否真实存在
  - 检查是否有竞争者填补了该空白
  - 检查空白声明是否有文献支撑
  - 评估空白的：
    - 真实性（Truthfulness）：0-10
    - 重要性（Importance）：0-10
    - 新颖性（Novelty）：0-10
    - 可研究性（Researchability）：0-10

- [ ] **生成空白验证报告**：
  - 每个空白的验证结果
  - 竞争者分析
  - 空白真实性评分

### Step 4: 科学假设评估 (约1小时)

- [ ] 提取论文的核心科学假设
- [ ] 评估假设的：
  - 可证伪性（Falsifiability）：0-10
  - 新颖性（Novelty）：0-10
  - 可行性（Feasibility）：0-10
  - 重要性（Importance）：0-10

- [ ] 检查假设是否有：
  - 明确的反证路径
  - 实验设计支撑
  - 方法论支撑
  - 理论支撑

- [ ] **生成假设评估报告**：
  - 假设提取
  - 各维度评分
  - 改进建议

### Step 5: 解决方法评估 (约1小时)

- [ ] 分析方法论的：
  - 科学性（Scientificity）：0-10
  - 先进性（Advancedness）：0-10
  - 可行性（Feasibility）：0-10
  - 可复现性（Reproducibility）：0-10

- [ ] 与现有最佳方法（SOTA）对比：
  - 方法是否优于现有方法？
  - 性能指标对比
  - 计算资源对比
  - 适用场景对比

- [ ] **生成方法评估报告**：
  - 各维度评分
  - SOTA对比
  - 改进建议

### Step 6: 文献引用质量评分 (约0.5小时)

- [ ] 评估引用文献的：
  - 权威性（Authority）：0-10
  - 相关性（Relevance）：0-10
  - 时效性（Timeliness）：0-10
  - 多样性（Diversity）：0-10
  - 完整性（Completeness）：0-10

- [ ] 识别缺失的重要文献：
  - 最近3年重要文献是否引用？
  - 关键领域重要文献是否引用？
  - 不同学派文献是否引用？
  - 不同方法文献是否引用？

- [ ] **生成引用质量报告**：
  - 各维度评分
  - 缺失文献列表
  - 补充建议

### Step 7: 综合评分与报告 (约0.5小时)

- [ ] 综合评分（加权平均）：
  - 引用验证：30%
  - 空白验证：20%
  - 假设评估：20%
  - 方法评估：20%
  - 引用质量：10%

- [ ] 确定推荐：
  - >= 90: ACCEPT
  - >= 75: ACCEPT with minor revisions
  - >= 60: MAJOR_REVISION
  - < 60: REJECT

- [ ] 生成最终报告：
  - 总体评分
  - 推荐意见
  - 具体问题
  - 改进建议

## 审查原则

### 1. 诚实优于取悦

- 不说用户想听的话，说真实的话
- 不确定时标明不确定性
- 具体引用优于模糊概括
- 清楚标记"这是事实" vs "这是我的推断"

### 2. 精确优于模糊

- 每个断言必须有可验证的来源
- 每个评分必须有明确的理由
- 每个建议必须是具体的、可执行的

### 3. 批判优于附和

- 对假设做批判性评估
- 检查每个论点的逻辑严谨性
- 检查每个结论的支撑证据
- 检查每个方法的科学性和先进性

### 4. 凡引必验

- 每篇参考文献必须验证其引用理由
- 每个引用上下文必须准确
- 每个引用理由必须合理

### 5. 空白的独立性验证

- 不可信任论文自己声称的空白
- 必须独立检索最新文献验证
- 必须检查是否有竞争者填补了空白

## 时间估算

| 步骤 | 时间 | 备注 |
|------|------|------|
| Step 1: 文献下载 | 1-2小时 | 取决于参考文献数量和下载速度 |
| Step 2: 引用验证 | 2-3小时 | 每篇文献约5-10分钟 |
| Step 3: 空白验证 | 1-2小时 | 取决于空白数量和独立验证复杂度 |
| Step 4: 假设评估 | 1小时 | 取决于假设复杂度 |
| Step 5: 方法评估 | 1小时 | 取决于方法复杂度 |
| Step 6: 引用质量 | 0.5小时 | 自动化程度较高 |
| Step 7: 综合报告 | 0.5小时 | 汇总和格式化 |
| **总计** | **6-9小时** | 可根据时间调整深度 |

- `review-hypothesis-assessment.md` — 假设评估报告
- `review-method-assessment.md` — 方法评估报告
- `review-citation-quality.md` — 引用质量报告
- `review-final-report.md` — 综合报告
