# Paper Citation Health Scan (D8/D10a)

Scan all papers in `outputs/papers/` for citation bibliographic health metrics D8 (bib entries) and D10a (cite-to-bib match %).

## Trigger
- Run periodically (cron) as a library-wide health check
- Run manually before major paper submissions

## Prerequisites
- Python 3.x available
- Papers stored under `/media/yakeworld/sda2/Synthos/outputs/papers/` (or configured path)

## Scan Logic

### Paper Discovery
1. Walk `outputs/papers/` for all `paper.tex` files
2. **Deduplication**: prefer `01-manuscript/paper.tex` over root `paper.tex`
3. Group by parent directory to avoid counting subdirectory copies

### Bib Resolution (per paper)
1. Priority 1: `{paper_dir}/06-references/references.bib`
2. Priority 2: `{paper_dir}/references.bib` (skip shared top-level `references.bib`)
3. Priority 3: any other `.bib` file at paper root (skip shared)
4. Fallback: none → D8=0

### Metrics
- **D8**: Count of `@Entry` lines in the paper's own `.bib` file
- **Cite keys**: Extract from `\cite{...}`, `\citep{...}`, `\citet{...}`, `\nocite{...}`, `\fullcite{...}`
- **Orphans**: Cite keys not found in any bib key (name matching with underscore/hyphen normalization)
- **Zombies**: Bib keys not cited anywhere in the paper
- **D10a**: `(total_cites - orphans) / total_cites * 100` (100% if no cites)
- **Compiled**: ✅ if `.bbl` file exists in paper dir, ❌ otherwise

### Problem Threshold
- D10a < 100% → problem
- D8 < 30 → problem (low reference count)

## Common Issues Found

1. **Missing .bib files**: Many papers in this library have `paper.tex` with `\cite{}` commands but no `.bib` file at all. This is the dominant failure mode.
2. **Shared bib confusion**: A shared `references.bib` at the top level of `outputs/papers/` contains 47 entries (AI/ML meta-references). Individual papers with their own bibs should NOT use this shared file.
3. **<label> and jabref-meta**: Some bib files contain JabRef metadata entries that match poorly.
4. **Uncompiled papers**: Many papers lack `.bbl` compilation output.

## Pitfalls

### Empty lines in scan output break report parser
The scan script output has empty lines after the header. When building report parsers, use `continue` on empty lines, not `break`. A `break` on first non-`|` line stops at the blank line between header and table.

### JabRef artifacts cause false positives
Some .bib files generated by JabRef contain `<label>` entries and `jabref-meta: databaseType:bibtex;` metadata that don't match any `\cite{}` key. These are not real orphans/zombies.

### `<label>` and `lamport94` phantom cite
Some bib files contain `lamport94` (auto-reference to LaTeX manual) that is NOT a real citation key — a JabRef artifact. Causes false orphan detection.

### Duplicate paper directories
Some papers have `paper.tex` in both `01-manuscript/` and `09-manuscript/` subdirs. The scan treats each as a separate entry (e.g., `name/09-manuscript`). Intentional but should be noted when comparing across runs.
