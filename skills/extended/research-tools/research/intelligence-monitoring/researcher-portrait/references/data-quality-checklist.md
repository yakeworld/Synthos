# 商业计划书数据质量评估清单

> 实战日期：2026-05-30 | 场景：医学装备创新大赛商业计划书
> 关键教训：不要相信任何未经用户确认的"已发表"状态

## 数据质量层级

每项数据按可信度分为四级：

| 层级 | 标签 | 含义 | 使用方式 |
|:-----|:-----|:------|:---------|
| A | ✅ 确凿 | 有明确来源可追溯 | 可直接写入，标注来源 |
| B | 🟡 合理估算 | 基于已知数据的推算 | 写入时加注"预估" |
| C | ⚠️ 需核实 | 来源不足或存疑 | 写入时加注"据XX报告/据行业调研" |
| D | ❌ 需修正 | 确认有误 | 从文档中删除或修正 |

## 六大数据类别检查要点

### 1. 团队信息
- [ ] 全员姓名/电话/出生/邮箱/学历/毕业院校已确认
- [ ] 导师资格/职称/人才称号有来源
- [ ] 项目负责人背景信息来自官网或用户确认

### 2. 学术成果
- [ ] **每篇论文确认发表状态**：已发表？已投稿？已完成待投稿？
- [ ] 期刊名称与实际投稿期刊一致（非内部代码/范例名）
- [ ] T1/T2等内部质量评分不混淆为"已发表"
- [ ] 论文总数与正文列表一致（标题不低估）

### 3. 专利成果
- [ ] 专利总数已核对（仅ZL+CN号匹配可能漏）
- [ ] 已授权（ZL）和实审中（CN）分开标注
- [ ] 发明人排序正确（1=第一发明人）
- [ ] 实用新型和发明专利的"双报"现象已标注

### 4. 市场数据
- [ ] 每条市场数据标注来源（如"据XX报告"）
- [ ] 无据可查的市场规模改为定性表述
- [ ] 增长率/渗透率等具体数字必须有出处

### 5. 财务数据
- [ ] BOM成本标注"预估"或"基于XX经验"
- [ ] 营收预测标注"基于以下假设..."
- [ ] 融资方案标注"可协商"

### 6. 产品技术
- [ ] TRL等级标注"自评"或弱化为"算法级验证完成"
- [ ] 具体芯片型号（如ARM Cortex-A78）改为通用描述
- [ ] 集成程度据实描述（分模块验证≠全部集成交付）

## 常见陷阱

| 陷阱 | 表现 | 根因 | 预防 |
|:-----|:-----|:-----|:-----|
| **"已发表"误用** | 未发表的论文写为"已发表" | 把内部质量评估等同于发表 | 逐篇确认后再写 |
| **专利数低估** | HTML表格只解析了前半 | 正则匹配ZL/CN号漏行 | 用`<tr>`分组解析 |
| **市场数据虚构** | 写入无来源的具体数字 | 为填充内容而编造 | 无来源就改定性表述 |
| **政策引用模糊** | "据国家卫健委文件"无文号 | 无法确认具体政策 | 改为"据相关政策" |
| **产品规格绝对化** | 写具体芯片型号但未选定 | 以为设计已定型 | 改为"高性能ARM处理器" |
