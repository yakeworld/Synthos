## 原理层·文言

> 凡临新域，先观其天：天者，政也，势也。
> 一级七部定方向，二级省策布棋局，三级市规见落子。
> 再看行伍已占者何人，三看己身可接者何在，
> 方知进路、退路、避路、开路。
> 四步而已：政→局→接→建。

## 触发条件

加载此技能当用户：
- 收到新兴技术/产业领域座谈会/政策讨论邀请
- 需要了解某方向的国家→省→市政策体系
- 需要判断自身研究在新兴赛道中的定位和差异化空间
- 需要为政府决策建言提供数据支撑

**不适用**：已有明确研究方向需深入文献 → research-paper-search；需创意发散 → research-ideation；需系统综述 → systematic-review；需写提案 → political-proposal

## 四步扫描法

```
Step 1: 政策金字塔（政）
  ├── 国家层 → 部委文件/总理令/专项规划
  ├── 省级层 → 省科技厅/卫健委/发改委文件
  ├── 市级层 → 本市已有政策/空白判断
  └── 相关配套 → 医保定价/药监审批/标准体系
↓
Step 2: 竞争格局（局）
  ├── 已挂牌基地/示范区/临床中心有哪些？
  ├── 标杆机构的模式是什么？（谁牵头+谁合作+做什么）
  └── 你所在地区的排名和窗口判断
↓
Step 3: 学术挂接（接）
  ├── 团队已有成果中哪些可直接挂钩？
  ├── 挂接路径描述（具体到技术模块）
  └── 差异化定位（给什么空白画什么框）
↓
Step 4: 策略建议（建）
  ├── 近期动作（0-3月）
  ├── 中期目标（3-12月）
  └── 会议发言要点
```

## Step 1: 政策金字塔检索

### 1.1 检索顺序

```
国家部委 → 省级 → 市级 → 配套政策（医保/审批/标准）
```

### 1.2 国家政策检索技巧

| 搜索词策略 | 说明 |
|-----------|------|
| `[领域] + 实施意见/管理办法/指导意见` | 查找部委核心文件 |
| `[领域] + 2025/2026 + 政策` | 查找最新文件 |
| `国务院 + [领域] + 发展` | 从gov.cn检索 |
| `科技部/工信部/卫健委 + [领域]` | 按部委检索 |

**推荐检索方式**：
- 使用 DuckDuckGo HTML 模式（`html.duckduckgo.com/html/?q=...`），对中文政策检索兼容性好
- 优先抓取 gov.cn / miit.gov.cn / ndrc.gov.cn 等官方网站
- 用 Python/curl 直接请求官方页面，避开 JS 渲染问题
- 备选：Bing 搜索（需设置 `Accept-Language: zh-CN,zh;q=0.9`）

### 1.3 省级政策检索

- 搜索 `[省名] + [领域] + 科技创新/产业发展/产学研`
- 关注：省科技厅政策包、省发改委规划、省政府工作报告
- 核心信息提取：量化目标（营收/产值）、布局规划（一核N基地）、扶持力度（补助金额）

### 1.4 竞争格局检索

- `[领域] + 示范/基地/临床中心`
- 搜索时重点关注：授牌机构、合作企业、已开展业务、覆盖技术范围
- 提取每个标杆的 **模式总结**（谁+谁合作+做什么）

## Step 2: 技术挂接分析

### 2.1 挂接逻辑模板

```
团队已有成果 → 技术模块X → [领域]的分支Y → 应用场景Z
```

### 2.2 典型挂接路径类型

| 路径类型 | 描述 | 示例 |
|----------|------|------|
| **输出型** | 现有成果作为新领域的核心技术模块 | Kappa角校准 → 眼控BCI精度提升 |
| **协同型** | 现有成果与新领域方案合并成多模态方案 | 瞳孔追踪 + EEG → 多模态BCI |
| **拓展型** | 现有成果在**特定疾病亚型**中天然适用 | BPPV复位动力学 → 前庭-BCI闭环 |
| **工具型** | 现有设备/平台直接为新领域提供基础设施 | 已有眼动仪 → BCI系统的眼动通道 |

#### 2.3 成果表述的精确性原则（⚠️ 关键）

> 在政策咨询/座谈会的学术挂接陈述中，**用语精确度直接影响可信度**。
> "已发表"与"已完稿"是重大区别——说错一次，后面所有的数据都会被质疑。

| 实际状态 | 正确的描述用语 | 错误的描述用语 |
|----------|---------------|---------------|
| 已正式发表（有DOI/卷期页码） | "已发表于[期刊名]" | "已完稿" |
| 已完稿，未投/在审 | "已完稿""已完成手稿" | "已发表""已发表论文" |
| 已在arXiv预印本 | "已发布预印本于arXiv" | "已发表" |
| 有算法代码无手稿 | "已开发完成[算法]" | "已有论文" |
| 有系统/平台无论文 | "已建成[系统/平台]" | "已有学术成果" |

**不要模糊表述。** 如果手稿未发表，就说"已完成手稿""待发表手稿"——用精确的语言赢得信任，而不是模糊的用词暴露不确定性。

### 2.4 🔄 强度转述策略（关键技巧）

当团队在手稿阶段（无正式发表）但希望展示实力时：

```
不要只讲："我们有N篇手稿。"
而要讲："我们有完整的[系统名/方法论]——它经过N轮迭代验证，
        产出了这M篇手稿。系统/方法本身就是实力的证明。"
```

**原理**：从"成果叙事"切换到"**能力叙事**"。手稿是阶段性的，系统/方法/平台是持久性的。政策制定者更关心"你持续产出的能力"而非"你现在有多少篇论文"。

**示例**（以Synthos为例）：
> "我们的科研操作系统经过53轮自进化迭代，建立了121个可执行技能和四层质量闸门。这个系统已产出64篇完整手稿，其中有3篇已提交在审。系统本身证明我们有持续生产高质量科研成果的工程化能力。"

这种转述适用于任何有方法论沉淀但发表数量尚在积累期的团队。

### 2.5 空白定位原则

> **不要重复赛道。找到别人还没做的细分方向，然后定义它就是你的方向。**
> 标杆做了什么 → 他们没做什么 → 你的成果最适合做什么 → 这三者的交集就是定位。

## Step 3: 策略建议框架

### 3.1 近期（0-3月）

- [ ] 梳理团队现有相关设备/数据/成果清单
- [ ] 搜索可合作的企业/机构
- [ ] 在座谈会/讨论会中提出差异化方向

### 3.2 中期（3-12月）

- [ ] 申报省级/国家级专项课题
- [ ] 争取纳入省级布局规划
- [ ] 推动签署企业战略合作协议
- [ ] 建立特色方向临床研究单元

### 3.3 会议发言结构

```
一、行业判断（30%）
   - 国家正在加速什么（引用政策）
   - 兄弟省市在做什么（引用标杆）
   - 窗口期判断
二、已有基础（30%）
   - 团队/单位已具备的条件
   - 与议题方向的具体挂钩点
三、具体建议（40%）
   - 差异化定位建议
   - 可立即启动的事项
   - 需要的支持
```

## 陷阱

- **不要只搜百度**：百度对政策检索限制多，DuckDuckGo HTML模式 + 定向抓取 gov.cn 更稳定
- **不要遗漏医保/审批政策**：定价和准入政策往往决定临床应用的商业化可行性
- **不要模仿标杆**：同质化竞争无意义，找空白比跟风更重要
- **不要只讲大方向**：挂钩路径必须具体到技术模块和成果名称
- **注意引用时效性**：政策文件标注成文日期，会议中引用要说明"截至X年X月"
- **成果用语不精确**："已发表"和"已完稿"不可混用。政府座谈会上说错一次，后面所有数据都会被质疑。务必按[2.3 成果表述的精确性原则](#23-成果表述的精确性原则关键)严格区分
- **只列成果不亮系统**：手稿数量的震撼力有限，但"系统+方法论"的叙事更有力量。务必使用[2.4 强度转述策略](#24-强度转述策略关键技巧)将听众注意力从"有多少篇发表"转向"有什么样的产能"
- **不要虚构数据或进展**：政策座谈会上被追问细节时，编造的东西一定会露馅。宁可缩小范围说"已完成"，也不要夸大说"已发表"

## 政策建议书写作规范（⚠️ 用户偏好）

用户在撰写政府座谈会意见稿时有两个硬性偏好，任何意见稿类产出都必须遵守：

### 原则1：一个建议原则
> **只提一个核心建议**，不要分散。一个清晰、聚焦的建议比五个零散的建议有力得多。

示例（脑机接口座谈会）：不是同时提"增加眼动-前庭方向""建设验证平台""建立攻关清单"三个建议，而是统一为**一个**核心建议——"将混合脑机接口作为温州BCI发展特色方向"，其他内容都围绕这个建议展开。

### 原则2：城市战略视角，不突出具体单位名称
> 意见稿应从**城市整体战略**角度论述"温州为什么要搞"，而不是从"INSTITUTION_NAME_PLACEHOLDER/某某团队"角度论述"我们需要什么"。

具体规则：
- 把"INSTITUTION_NAME_PLACEHOLDER"替换为"温州临床科研力量"或"温州临床资源"
- 把"我们团队有XX"替换为"温州已具备XX"
- 把关口从"我要资源"转为"温州的优势是什么、为什么应该布局这个方向"
- 最终产出是"温州应该做什么"的论述，而非"我们团队需要被纳入"的申请

### 文件产出规范
意见稿产出通常包含三个层次，按需生成：
- **发言稿**（speech）：会上实际念的，口语化，5-7分钟
- **修改建议**（scheme-suggestions）：对照初稿逐条标注改动
- **完整建议意见稿**（independent）：独立成文的正式建议书，Markdown格式即可，不需要PPT
