# 论文研究空白审计模板（Research Gap Audit）

> 基于膜性SCC重建论文实战（2026-05-31），G1-G7系统性空白推导。
> **铁律**：此文档存放在论文自身的 `09-background/` 目录中，随论文版本管理。

---

## 何时执行

论文进入 **投稿包就绪** 状态后，**下一次迭代前**立即执行。此时论文的所有结果/方法/Discussion已定型，是推导后续方向的最佳时机。

## 方法论：四象限扫描

从四个维度系统扫描未答问题：

```
          ┌──────────────────────────────────────────┐
          │          已完成论文（结果+方法+讨论）        │
          └──────────────────────────────────────────┘
                      ↓ 四象限扫描
    ┌──────────────────┬──────────────────┐
    │ ① 方法学局限     │ ② 数据规模不足   │
    │ 现有方法做不到什么？ │ n太小/模态单一/缺对照 │
    ├──────────────────┼──────────────────┤
    │ ③ 结果延伸       │ ④ 临床转化      │
    │ 发现了什么但没深挖？ │ 对诊断/治疗/预后的意义 │
    └──────────────────┴──────────────────┘
```

每个象限至少产生1个空白。

---

## 模板：每个空白的结构

```markdown
### G{N}. {空白标题（简明一句话）}

**象限：** [方法学/数据/延伸/转化]

**空白：** 目前已知X，但Y仍未被探索/建模/量化。

**科学问题：**
- 具体可问的问题1？
- 具体可问的问题2？

**假设：** 具体、可检验、可证伪的因果/关系声明。

**验证方法：** 用什么数据、什么分析、什么统计检验。

**优先级：** ⭐立即推进 / 🟡待条件成熟 / 长期

**与本文的关系：** Discussion中如何引向此方向。
```

## 实战案例：膜性SCC论文G1-G7

完整版见 `membranous-scc-reconstruction/09-background/SCC-Research-Gaps.md`，这里摘要关键模式：

### 从 "局限性→空白" 的推导路线

| 论文局限性 | → | 研究空白 | 推导逻辑 |
|:----------|:---|:---------|:---------|
| n=6对骨膜对，leave-one-out预测失败 | → | G3: 更大样本的骨→膜预测模型 | 论文已证明"个体差异＞系统偏差"→需要n≥20 |
| 只有静态CT，无动态数据 | → | G3: 头动加速度下的动态骨膜偏移 | 膜迷路悬浮于trabeculae→加速度可能引起相对位移 |
| 只有μCT/MRM，无临床CT | → | G4: SCC参数作为PD生物标志物 | PD患者CT已可获取，SCC管线全自动 |
| 只量化了偏差，没做应用 | → | G1: 个体化BPPV复位 | 骨膜偏差→个体解剖差异→复位路径优化 |

### 最重要的导出公式

**从论文Discussion末段的投射框架**，直接转化为新的可检验假设：

$$
r_{MEM}(s) = r_{BONY}(s) + \mathbf{d}(s)
$$

这个等式的右边三项分别对应三个未来方向：
- $\mathbf{d}(s)$ 本身 → 更大样本的偏差场统计建模（G3）
- $r_{BONY}(s)$ 作为输入 → 预测 BPPV 个体解剖易感性（G7）
- $r_{MEM}(s)$ 作为输出 → 将骨性几何映射到膜性功能结构（G1个体化仿真）

## 优先级判断标准

| 优先级 | 条件 | 行为 |
|:------:|:-----|:-----|
| ⭐ 立即推进 | 数据已有、代码就绪、无伦理障碍、与团队方向一致 | 创建paper目录+paper.md，排入待办 |
| 🟡 待条件成熟 | 需新数据/合作/审批，但方案清晰 | 记录在案，标注缺少的条件 |
| 长期 | 需跨物种/硬件开发/>1年随访 | 只做摘要，不展开 |
| 已覆盖 | 已有其他论文在推进此方向 | 标记existing=[论文名] |

## 与 hypothesis-generation 技能的关系

本审计产出 **研究空白**（GAP层），是 `hypothesis-generation` 技能的输入。hypothesis-generation 将空白转化为形式化假设（含预测、反证条件、竞争假说、实验设计）。两者分工：

```
本审计（GAP）              hypothesis-generation（HYP）
───────────────            ───────────────────────
空白描述                   形式化假说（含预测+反证条件）
科学问题                   可检验假设
验证想法                   验证方案设计+代码（fast-track）
优先级判定                  两轨路由（fast/plan）
论文目录组织                积分入假说库
```

推荐流程：审计产出空白文档 → 加载 hypothesis-generation 技能 → 每空白生成≥1形式化假说 → 验证/排程。

## 反模式（不要这样做）

| 反模式 | 问题 | 正确做法 |
|:-------|:-----|:---------|
| 放在 `~/桌面/article_todo/` | 与论文脱离版本管理，多论文共享一个位置导致混乱 | 放在各论文的 `09-background/` 下 |
| 空泛的方向（如"需要更多研究"） | 不可检验，无科学价值 | 每个空白必须带可检验假设和验证方法 |
| 只列方向不标优先级 | 读者/审稿人不知道哪些是认真的 | 使用 ⭐/🟡/长期 三级标注 |
| 与本文无关的方向 | 审稿人会问"这和你论文有什么关系" | 每个空白说明与本文的推导关系 |
