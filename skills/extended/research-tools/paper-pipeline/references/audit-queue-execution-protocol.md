# Audit Queue Cron Job — Execution Protocol

## Purpose
This document captures the execution protocol for the paper-quality-audit cron job, which reads AUDIT_QUEUE.md, audits papers in the queue, and updates gate_status + state.json.

## Execution Flow (from cron job spec)

1. Read `AUDIT_QUEUE.md` — `head -5 /media/yakeworld/sda2/Synthos/outputs/papers/AUDIT_QUEUE.md`
2. Process one paper at a time:
   a. Check `quality_score` (if 0, skip)
   b. Check 4 reports in `07-quality/` directory
   c. If all PASS and fixes complete → remove from queue
   d. If P0_WAITING_USER → mark BLOCKED
   e. Update gate_status: VERIFIED/BLOCKED/IN_PROGRESS
   f. Update state.json: gate_status + notes
3. If queue empty, output [SILENT]

## Quality Gate Standards

- **6-domain reports**: Universal six domains + specialty-specific + reference audit + inspector report
- **Reference audit**: D8 (format) + D10a (traceability)
- **Numeric verification**: G7 (reproducibility)
- **Pass threshold**: quality_score ≥ 0.85

## State Update Pattern

When updating gate_status in state.json:
```json
{
  "gate_status": "BLOCKED",
  "quality_score": 73,
  "notes": {
    "audit_2026-06-29": "Description of audit findings..."
  },
  "last_modified": "2026-06-29 22:00"
}
```

## Audit Queue Update Pattern

When updating AUDIT_QUEUE.md:
- Use 🔒 BLOCKED for papers that need independent action
- Remove verified papers from the table entirely (mark ✅ VERIFIED with date + summary)
- Keep IN_PROGRESS papers with 🔄
- Papers stay as "待审计" until a cron job processes them

## Pitfalls

1. **Don't trust state.json gate_status alone** — Always cross-check 07-quality/ for actual report files
2. **Legacy quality-report.md is NOT a standard audit report** — It's a D8/D10a scan output, not one of the 4 required reports
3. **step_quality_review.md is NOT a standard report** — It's generated by the G7 step in the pipeline, not an independent audit
4. **D8=0 means no references.bib exists** — This is the most severe issue; requires full reference体系建设
5. **State.json modification race condition** — Sibling subagents may also modify state.json; always read before writing
6. **AUDIT_QUEUE.md also has race conditions** — Multiple cron runs may update the same file; read before writing

## Execution Notes

- Queue processes one paper per cron run
- Highest priority: P0 (public dataset) > P1 (BPPV) > P2a (eye algorithm) > P2b (eye with data) > P2c (eye other)
- Within each priority, sort by quality_score descending
- Papers with quality_score=0 are skipped (no score yet)
- The queue lists ~91 papers, most with "待审计" status
