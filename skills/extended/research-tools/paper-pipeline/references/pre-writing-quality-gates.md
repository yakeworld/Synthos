# Pre-Writing Quality Gates (P-1阶段节点闸门)

> 论文写作前的科学问题定义阶段，每个节点设独立闸门。不过不前进。

## 流程图

```
逐问文献检索
    │
    ▼
┌─────────────┐      FAIL
│  Gap Gate   │───────→ 回到Q2/Q3重新定位
│  (节点闸门)  │
└──────┬──────┘
       │ PASS
       ▼
┌─────────────┐      FAIL
│ Hypothesis  │───────→ 回到Q4重新形成
│ Gate        │
└──────┬──────┘
       │ PASS
       ▼
┌─────────────┐      FAIL
│ Method Gate │───────→ 回到Q5重新设计
└──────┬──────┘
       │ PASS
       ▼
┌─────────────┐      FAIL
│ Experiment  │───────→ 回到Q6重新设计
│ Gate        │
└──────┬──────┘
       │ PASS
       ▼
    P0 前置审计
```

## Gap Gate

| 检查项 | 方法 | 通过标准 |
|:-------|:-----|:---------|
| 空白真实存在 | `ask "这个空白在源文件中有证据支持吗？"` | 至少2个独立来源确认 |
| 有理由填 | `ask "填上它能解决什么具体临床/理论问题？"` | 论证非"为做而做" |
| 不是"没人做过" | `ask "没做过的原因是什么？不可行还是没人想到？"` | 有明确的技术/理论障碍 |
| 形式化完整 | Gap陈述包含(1)已知(2)未知(3)意义 | 三要素齐全 |

**淘汰标准**: 任何一项不通过 → 回到Q2/Q3

## Hypothesis Gate

| 检查项 | 方法 | 通过标准 |
|:-------|:-----|:---------|
| 可证伪 | `ask "什么数据/结果能拒绝这个假设？"` | 有具体的淘汰条件 |
| "If X then Y" | 假设有明确的形式 | 变量可操作化 |
| 有阈值 | Y有数量化标准 | ≥/≤/=/≠ 具体数值 |
| 不与其他结论矛盾 | `ask "这个假设与已知文献冲突吗？"` | 没有直接矛盾 |
| 有替代假设 | 至少列出H₂ | H₂有合理的可能性排序 |

**淘汰标准**: 任何一项不通过 → 回到Q4

## Method Gate

| 检查项 | 方法 | 通过标准 |
|:-------|:-----|:---------|
| 能回答假设 | 方法直接验证假设中的因果关系 | 方法↔假设映射明确 |
| 可行性 | 每个步骤有代码/数据/硬件支撑 | 无"黑箱"步骤 |
| 已有资产 | `source list` 检查已有代码和数据 | ≥80%步骤已有资产 |
| 缺失标记 | 缺失步骤有明确标记 | (待开发)/(需采购) |

## Experiment Gate

| 检查项 | 方法 | 通过标准 |
|:-------|:-----|:---------|
| 样本量合理 | 有统计学依据 | N≥30或power analysis |
| 指标匹配假设 | 实验测量的正是假设中的Y | 指标↔假设一一对应 |
| 控制组存在 | 至少有基线对比 | 不是单臂 |
| 成功/失败标准 | 有预先设定 | 不是事后编故事 |

## 与P1-P4质量门的关系

| 阶段 | 闸门 | 作用 |
|:-----|:-----|:------|
| P-1 | 节点闸门（本文件） | 确保科学问题正确 |
| P1-P3 | 无独立闸门 | 执行阶段 |
| P4 | G1-G7 + 7维SCI评分 | 确保论文产出质量 |
| 后P4 | 递归进化 | 确保知识迭代 |
