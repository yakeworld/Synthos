# 2026-05-26 会话经验总结

## 实验代码验证优先于双质检

发现 LLM 论文数值虚高时，正确的修复顺序：
1. 审查实验代码（可能有 bug）
2. 修复 bug
3. 运行实验获取真实数据
4. 对比论文声称 vs 实验结果（检查方向一致性）
5. 用真实数据重写论文
6. 再跑双质检

详见 `references/experiment-code-verification.md`

## 多论文共享 NotebookLM 项目

- 一个 NotebookLM 项目可以产出多篇同主题论文
- 上传时用独立命名：`{dir-name}-v{N}.pdf`
- 用 `--title` 标注完整论文名+版本

## AKNE 项目评估

AKNE (Autonomous Knowledge Network Engine) 已基本被 Synthos + NotebookLM 取代。19个源目录含 1,144 个 .md 文件 (17MB)，为 LLM 生成的 wiki 笔记。建议方案：
- 用 rclone 同步到 Google Drive（已配 client_id，需 OAuth 授权）
- 按需导入 NotebookLM
