# CJK FTS5 全文检索修复 — 2026-05-27

> 问题: `fact_store(action='search', query='记忆')` 返回空结果
> 根因: SQLite FTS5 默认 `unicode61` tokenizer 把整段中文视为一个 token（如`"记忆功能测试"`），单字/短词无法匹配

## 排查路径

| 步 | 操作 | 发现 |
|:--:|:-----|:------|
| 1 | `select * from facts` | 数据存在，非空 |
| 2 | `facts_fts MATCH '记忆'` | 空结果 |
| 3 | `facts_fts MATCH 'Holographic'` | ✅ 英文命中 |
| 4 | `fts3tokenize(unicode61)` 测试 | CJK 串 `"记忆功能测试"` → 单 token |
| 5 | `LIKE '%记忆%'` | ✅ 命中 |

## 修复方案

改 `plugins/memory/holographic/retrieval.py` 中 `_fts_candidates()` 和 `_tokenize()`：

### `_fts_candidates()` — CJK 检测 + LIKE 分支

```python
_CJK_RE = re.compile(r'[\u4e00-\u9fff\u3400-\u4dbf\uf900-\ufaff]')

if self._has_cjk(query):
    # LIKE-based search: WHERE content LIKE '%query%'
    # 每个 LIKE 结果给 fts_rank=0.5（中性），Jaccard reranking 做实际评分
    ...
else:
    # FTS5 MATCH path (original)
    ...
# 兜底: FTS5 空结果 → 也退到 LIKE（补嵌ASCII字符在CJK中的情况）
```

### `_tokenize()` — CJK 字符级拆分

```python
# 原: 仅 whitespace split
# 新: whitespace split + CJK 单字拆分
tokens = set()
for word in text.lower().split():
    cleaned = word.strip(".,;:!?\"'()[]{}#@<>")
    if cleaned:
        tokens.add(cleaned)
cjk_chars = set(_CJK_RE.findall(text))
tokens.update(cjk_chars)  # 让 Jaccard 能评分中文单字
```

### 核心原理

```
unicode61 tokenizer: "记忆功能测试" → ["记忆功能测试"] (单token)
LIKE '%记忆%':                                   → 子串匹配 ✅ (但无排名)
LIKE + Jaccard 字级评分: 拆成 {记,忆,功,能,测,试} → 正确排名 ✅
```
