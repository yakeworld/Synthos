# 征求意见时的预判输出协议

## 来源

2026-05-27 用户纠正：征求意见时必须附上对人类反应的预判。
"每次征询意见的时候，在最后，你要给出人类可能回答的这个预判。"

## 铁律

任何时候呈现选项（哪怕只有1个选项），输出格式必须为：

```yaml
方案A: ...
方案B: ...

🔍 推测: 我预测人类会选 [A/B], 因为 [推理: 历史模式/成本/用户偏好...]
```

## 不中断原则

呈现选项+预测后继续工作。预测不是"等确认"——是"错了用户纠正，对了继续前进"。

## 向量子对照表

| 输出置信度 | 行为 | 预测要求 |
|:----------|:-----|:---------|
| ≥80% | 直接执行 | 末尾推理链 |
| 60-80% | 推荐方案+附纠正入口 | ✅ 必须附预测 |
| <60% | 给选项+推荐 | ✅ 必须附预测 |

## 禁止模式

- ❌ 呈现选项后只说"您觉得如何？"——没预测
- ❌ 呈现选项后等回复——应该预测后继续
- ❌ 预测后不做——预测后继续执行预测的方案
