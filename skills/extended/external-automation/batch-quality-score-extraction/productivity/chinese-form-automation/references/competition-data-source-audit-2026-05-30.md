# 商业计划书数据来源审计 — 实战案例

> 2026-05-30 医学装备创新大赛长三角分赛
> 项目：便携式智能眼动诊断系统

## 审计动因

用户反馈："计划书一定要实事求是的。所有的数据都要有来源的，你要提供一个质量评估报告。"

## 发现的问题

| 问题 | 严重度 | 描述 | 修正 |
|:-----|:------:|:------|:-----|
| 学术成果虚报 | 🔴 P0 | "已发表SCI论文5篇"——实际均未发表 | 改为"已完成待投稿论文" |
| 论文数量低估 | 🟡 P2 | "5篇"但实际有22篇+ | 改为"22篇第一/通讯作者SCI" |
| 专利数量低估 | 🔴 P0 | "发明专利5项"——实际6项已授权+10项实审中 | 全量更新 |
| 市场数据无源 | 🔴 P0 | 市场规模/增长率等无引用 | 改为定性或加"据XX报告" |
| 竞品价格凭印象 | 🟡 P1 | 无来源标注 | 改为"据公开信息" |
| 产品规格过具体 | 🟡 P1 | ARM Cortex-A78芯片型号 | 改为"高性能ARM处理器" |

## 修正后效果

- 商业计划书v2: 数据准确度大幅提升
- 同步修正：研究者面貌 + 质量评估报告

## 关键教训

1. **所有"具体数字"必须逐项核实** — 即使是最自信的"5篇论文"
2. **本地论文目录才是论文状态的权威来源** — 不是memory也不是感觉
3. **网站数据优先于本地记忆** — 网站专利表28行 vs 我记忆的5项
4. **财务预测必须标注"预估值"** — 否则用户会认为你在编造
