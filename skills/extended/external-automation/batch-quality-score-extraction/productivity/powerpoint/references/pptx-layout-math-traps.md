## 页面标准

| 属性 | 值 |
|------|-----|
| 宽度 | 13.333 英寸（2772 EMT） |
| 高度 | 7.5 英寸（1560 EMT） |
| 比例 | 16:9 |

## 核心公式

**右边界 = 左偏移 + 宽度** 必须 ≤ 13.333
**底边界 = 顶部偏移 + 高度** 必须 ≤ 7.5

### 布局结构示例（双栏 PPT）

```
左栏:  x=0.6, w=4.5  → 右边界=5.1 ✓
右栏:  x=5.3, w=2.6  → 右边界=7.9 ✓ (col=0)
       x=8.1, w=2.6  → 右边界=10.7 ✓ (col=1)
       x=10.9, w=2.6 → 右边界=13.5 ✗ 超出!
       x=10.7, w=2.6 → 右边界=13.3 ✓ (col=2, 压缩到13.333边缘)
```

### 行高计算

可用高度 = 7.5 - 标题区域(0.4) - 下划线(0.1) - 页脚(0.4) = **6.6英寸**

6个卡片3×2网格：(6.6 - 顶部留白0.3) / 3 = 2.1英寸/行（含间距）
实际卡片高2.0英寸，行距0.15英寸 → 最大底 = 1.3 + 2×2.15 + 2.0 = **7.6**（压7.5边缘）

### 多列网格布局

**3列布局**（每格宽 = (可用宽 - 总间距) / 3）：
```
可用宽 = 13.333 - 左侧边距(0.6) - 左栏宽(4.5) - 右侧边距(0.3) = 7.933
每格宽 = (7.933 - 2×0.2间距) / 3 = 2.6
```

## 5. 验证脚本的幻灯片索引陷阱

**PPTX内部zip文件按字典序排列**：`slide1.xml`, `slide10.xml`, `slide2.xml`...。直接 `sorted(z.namelist())` 会导致 `slide10` 排在 `slide2` 前。

**正确做法**：按数字提取页码，不要依赖文件名顺序。

```python
import re
with zipfile.ZipFile('file.pptx', 'r') as z:
    slides = sorted(
        [f for f in z.namelist() if re.match(r'ppt/slides/slide\d+\.xml', f)],
        key=lambda x: int(re.search(r'slide(\d+)', x).group(1))
    )
```

## 常见陷阱

1. **x + w > 13.333**：最后一列最易超。`col = max_col` 时校验 `x + w <= 13.333`
2. **y + h > 7.5**：最后一行最易超。`row = max_row` 时校验 `y + h <= 7.5`
3. **enumerate + tuple unpack 错位**：`for i, (a, b, c) in enumerate([...])` 中 `enumerate` 产生 `(i, (a,b,c))`，解包成 `(i, a, b, c)` 4个值，但 `for i, (a, b, c)` 期望3个。**去掉enumerate或手动解包**：
   ```python
   # 错误：
   for i, (ypos, title, lines) in enumerate(data):  # ValueError: too many values to unpack
   
   # 正确1：去掉enumerate
   for (ypos, title, lines) in data:
   
   # 正确2：手动解包
   for i, item in enumerate(data):
       ypos, title, lines = item
   ```
4. **bignum() 函数内部子形状溢出**：bignum 使用3个textbox（数字+后缀+单位），每个都要单独校验 `x+w` 和 `y+h`

5. **6项卡片 2×3 布局溢出陷阱**（⚡2026-06-11 瓯越英才 P3）：
   当右侧面板起点 x≥5.3、卡片宽≥2.5 时，`i//3, i%3` 产生3列，col=2 的右边界必超12.8"。
   ```python
   # 错误：6项 → 2行×3列 → col=2 溢出
   row = i // 3; col = i % 3
   x = 5.3 + col * 2.6  # col=2: x=10.5, right=10.5+2.5=13.0" ✗
   
   # 正确：6项 → 3行×2列 → 安全
   row = i // 2; col = i % 2
   x = 5.3 + col * 3.6  # col=1: x=8.9, right=8.9+3.3=12.2" ✓
   ```
   规则：右侧面板起始位置 ≥ 5.3 时，最大列数 ≤ 2。

6. **页码溢出陷阱**（⚡2026-06-11）：
   ```python
   # 错误：x=12.0+w=1.0=13.0" ✗  y=7.1+h=0.3=7.4" ✗
   tb(s, Inches(12.0), Inches(7.1), Inches(1), Inches(0.3), ...)
   
   # 正确：x=11.8+w=0.8=12.6" ✓  y=6.8+h=0.2=7.0" ✓
   tb(s, Inches(11.8), Inches(6.8), Inches(0.8), Inches(0.2), ...)
   ```

7. **全页背景豁免**：`rect(s, 0, 0, 13.333, 7.5)` 作为设计意图背景，安全区检查忽略。仅内容元素需遵守12.8"×7.0"。

8. **水平流式链溢出陷阱**（2026-06-11 P6）：
   多个节点+箭头水平排列时，最右节点极易溢出父卡片。
   ```python
   # 错误：4节点x1.55" + 3箭头 + start=0.85"
   # -> 最后右边界 0.85 + 3*1.85 + 1.55 = 7.95" > 卡片右界7.60" ✗
   xn = 0.85; step = 1.85; box_w = 1.55
   
   # 正确：缩小节点 + 紧凑步长
   xn = 0.70; step = 1.75; box_w = 1.45
   # -> 0.70 + 3*1.75 + 1.45 = 7.40" < 7.60" ✓
   ```
   规则：`start + (n-1)*step + box_w <= card_right`。

9. **嵌套卡片高度不匹配**（2026-06-11 P3）：
   当页面左右两栏高度不一致时，较短卡片内元素视觉"悬空"。
   ```python
   # 错误：左栏3.8"高，右栏延至6.6" → 左栏底部空洞
   rc(s, 0.6, 1.2, 4.8, 3.8, ...)  # 太短
   # 正确：左栏高度与右栏对齐
   rc(s, 0.6, 1.2, 4.8, 5.5, ...)  # 与右栏一致
   ```
   规则：同一页面左右两栏高度应一致（都到 ~6.7"），内部元素均匀分布。

## Helper 函数模板

```python
def safe_rect(s, l, t, w, h, fc, bc=None, bw=1):
    """安全矩形：自动校验边界"""
    if l + w > 13.333 or t + h > 7.5:
        print(f"WARNING: rect overflow at ({l},{t})+({w},{h})")
    return rect(s, l, t, w, h, fc, bc, bw)
```
