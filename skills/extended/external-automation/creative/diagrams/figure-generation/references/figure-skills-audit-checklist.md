# 作图技能全面审计清单

> 适用于任何作图技能体系的质量检查。按此清单执行审计，逐项标记通过/失败。

## 审计维度

### 1. 内容完整性（SKILL.md）

| # | 检查项 | 通过标准 |
|---|--------|----------|
| 1.1 | SKILL.md 长度 | ≥ 5000 字符（不含YAML frontmatter） |
| 1.2 | 方法层 | 有明确的执行流程描述 |
| 1.3 | 原理层 | 有"文言"概念层描述 |
| 1.4 | 触发条件 | 明确列出何时加载/不加载 |
| 1.5 | 验证清单 | 有执行前/执行后的检查清单 |
| 1.6 | 坑点/Pitfall | 至少有 3 个实战坑点 |
| 1.7 | 版本标记 | YAML frontmatter 有 version 字段 |

### 2. 引用文件完整性

| # | 检查项 | 通过标准 |
|---|--------|----------|
| 2.1 | references/ 存在 | 至少 5 个引用文件 |
| 2.2 | 无 Broken Reference | SKILL.md references 表中所有文件必须存在 |
| 2.3 | 无空引用文件 | 每个引用文件 ≥ 500 字符 |
| 2.4 | 无空目录 | 所有 subdirectories 非空 |
| 2.5 | BOUNDARY.md | 有实质内容（≥ 1000 字符） |
| 2.6 | IO_CONTRACT.md | 有实质内容（≥ 1000 字符） |
| 2.7 | EVIDENCE_SCHEMA.md | 有实质内容（≥ 1000 字符） |

### 3. 脚本完整性

| # | 检查项 | 通过标准 |
|---|--------|----------|
| 3.1 | scripts/ 存在 | 至少 1 个脚本文件 |
| 3.2 | 可执行性 | 脚本不依赖不存在的依赖 |
| 3.3 | 引用覆盖 | SKILL.md 中提到的每个 script 都实际存在 |

### 4. 去重检查

| # | 检查项 | 通过标准 |
|---|--------|----------|
| 4.1 | 跨技能重复 | 同一原语（如Pillow代码）不跨多个技能重复 |
| 4.2 | 内部重复 | SKILL.md 中同一内容不重复出现 |
| 4.3 | 色板一致 | 所有技能使用全局色板，不硬编码颜色 |

### 5. 空壳检测

| # | 检查项 | 通过标准 |
|---|--------|----------|
| 5.1 | 子技能空壳 | 无 SKILL.md < 2000 字符的子技能 |
| 5.2 | 引用为空 | 无引用文件仅有标题无内容 |
