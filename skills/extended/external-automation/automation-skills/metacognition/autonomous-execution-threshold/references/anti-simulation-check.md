# 反模拟执行检查清单

> 每次自主执行前，必须确认当前操作不违反反模拟铁律。
> 写入 autonomous-execution-threshold 作为执行前检查。

## 检查项目

| # | 检查 | 判定 | 通过条件 |
|:-:|:-----|:-----|:---------|
| 1 | 当前任务是否涉及技能执行？ | 是/否 | 否 → 跳过，是 → 继续 |
| 2 | pipeline_trace 中有 skill_view 记录？ | 有/无 | 必须有 `skill_view_called: true` |
| 3 | skill_version 已记录？ | 有/无 | 必须有版本号 |
| 4 | steps_executed 非空？ | 有/无 | 必须有已执行的步骤列表 |
| 5 | 如果是论文管线，G5引用验证已通过？ | 是/否 | 引用=PDF=候选 |

## 如果检查不通过

不要"补记录然后继续"——那是伪造证据。
应该：停止 → 重做原子（先 skill_view() 再执行）→ 再检查。

## 常见错误

| 错误 | 检查项 | 正确做法 |
|:-----|:-------|:---------|
| 直接write_file输出而不加载SKILL.md | #2 无skill_view记录 | skill_view() -> 按步骤执行 -> 输出 |
| 加载了SKILL.md但跳过输出步骤 | #4 steps_executed空 | 执行所有必选步骤 |
| 写了论文但引用不在候选列表中 | #5 G5不通过 | 从ACQ候选列表中选择引用 |
| 用模型知识写BibTeX而非真实搜索 | #5 引用无对应PDF | 先从PDF下载开始 |
