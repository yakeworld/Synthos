## Docx 填写核心教训

**问题**：写入 docx 合并单元格时覆盖了相邻标签文本。

**根因**：`python-docx` 中水平合并的单元格（gridSpan>1）共享同一个底层 XML `<tc>` 节点。`table.cell(ri, ci)` 对合并范围内的不同坐标返回同一个节点。

**正确做法**：不清除段落内容，直接用 `cell.paragraphs[0].text = val` 增量写入。

对于多行内容（工作经历、项目清单），写在合并区的 anchor 单元格（通常是 col=0）。

### 常见行布局模式

| 行 | 标签 | 值 |
|:---|:-----|:----|
| Row 0 | col0,2,4,6=标签（独立cell） | col1,3,5,7=值（独立cell） |
| Row 1-5 | col0-1=标签(span2) | col2-4=值(span3) | col5=标签 | col6-7=值(span2) |
| Row 6-8 | col0=标签 | col1-7=值(span7) |

```python
# 正确写法示例
set_cell(table, 0, 1, '杨晓凯')   # 独立cell → 安全
set_cell(table, 1, 2, '专业值')   # span3的anchor在col2
set_cell(table, 6, 1, '多行经历') # span7的anchor在col1
```

## Xlsx 填写要点

- `ws.cell(row, col, value=val)` — 不要写 MergedCell
- 表头行（前3行通常是合并的）直接跳过
- 数据从第4行开始
- `openpyxl` 写入比 `python-docx` 简单直接
