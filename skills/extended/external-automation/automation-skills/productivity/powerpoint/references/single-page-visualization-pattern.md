# PPTX Generation — 单页全景图模式 (2026-06-08)

**Date discovered:** 2026-06-08  
**Class of tasks:** 复杂信息单页压缩

## 目标

将多页信息（如14页PPT）压缩为一张高密度PPT全景图，用于汇报场景。

## 单页布局

```
+--------------------------------------------------+
| Header: 系统名称 + 版本 + 关键指标 (0.06in)      |
+--------------------------------------------------+
| Row 1: 系统定位 + 数字基座 (1.05in)              |
+--------------------------------------------------+
| Row 2: 三层架构 (认知层 | 执行层 | 进化层)        |
|        各1.95in宽, 1.95in高                      |
+--------------------------------------------------+
| Row 3: 宪法架构(左40%) + 技能体系(右60%)         |
|        宪法: 层级表 + 不可修改条款                |
|        技能: 3行表格 (甲/乙/丙)                   |
+--------------------------------------------------+
| Row 4: 论文管线流程 (6步骤, 水平排列)             |
|        每个步骤含名称+描述，箭头连接              |
+--------------------------------------------------+
| Footer: 核心优势总结 (1行, 全宽)                 |
+--------------------------------------------------+
```

## 执行步骤

1. 确定单页要覆盖的**6-8个核心板块**
2. 定义颜色方案：NAVY背景, TEAL分割, GOLD强调, 3列不同色
3. 每个板块用shp()画卡片，tb()写文本
4. 小字号(sz=9-11)，行间距紧凑
5. 用shp()画箭头连接流程步骤
6. 底部用NAVY背景条做核心优势总结

## 陷阱

- **不要试图塞入所有细节**：单页是概览，详细内容由多页PPT承载
- **字号不能太小**：最小9pt，保证投影仪可读
- **颜色不能太杂**：最多3-4种颜色，保持专业感
- **文本要精简**：每段≤3行，每行≤60字
- **使用系统PPT的配色**：从现有PPT提取NAVY/TEAL/GOLD palette，保持一致性
