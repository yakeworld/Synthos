# Zhihu Article Template

## Title Formula

《我开源了一个[一句话项目描述]——[吸引人的hook]》

Examples:
- 《我开源了一个能自我进化的AI科研系统——碳硅共生从哲学到代码》
- 《我开源了一个AI文档助手——让每个开发者少写一半注释》

## Structure

### 一、缘起：一个[身份]的「[问题]」
- 用具体数字说明问题（年门诊量6万、每月5万篇论文等）
- 引发共情的困境描述
- 引出核心问题：能不能用AI解决它？

### 二、[核心概念]：不是比喻，是事实
- 用三层次递进：能力互补 → 认知联觉 → 进化体
- 核心洞察（不是科幻，是可以运行的开源项目）

### 三、[项目名]：[如何设计的]
- 系统架构图（ASCII art）
- 核心模块/原子列表（表格）
- 设计原则（宪法/原则列表）

### 四、[关键特性/最有趣的部分]
- 这个项目和其他项目最不一样的地方
- 具体数据/成果（表格）
- 技术深度（不浮于表面）

### 五、为什么是开源的？
- 基础设施必须开源
- 智能体也可以参与
- 贡献流程说明

### 六、数据与成果
- 关键指标表格
- 对比同类项目

### 七、[愿景]：不是终点
- 更大的图景
- 长期的愿景

### 八、下一步
- 项目地址
- 对不同人群的价值（研究者/开发者/好奇者）
- 未来计划

## Tone
- 第一人称叙事
- 具体数字支撑
- 技术深度但避免晦涩
- 结尾开放，鼓励参与

## End Matter
```
*项目：[名称] | 协议：MIT | 作者：[身份]*
*讨论请到 GitHub Discussions：[链接]*
```
